# 功能说明文档 — 自媒体内容生产系统

> **版本**: v1.0 | **更新日期**: 2026-05-09
> **来源**: `00-项目/视频的自媒体账号.txt` 对话记录整理

---

## 一句话定义

**旅行自媒体运营者** 需要 **一套AI辅助的内容生产与运营系统** 来解决 **从对标分析、视频制作、多平台发布到数据迭代的全链路效率问题**。

---

## 系统架构：统一主干 + 可插拔模块

```
┌─────────────────────────────────────────────────────┐
│                 统一任务管理主干                       │
│  （任务创建 → 状态跟踪 → 模块调度 → 数据流转 → 归档）    │
├──────┬──────┬──────┬──────┬──────┬──────────────────┤
│ 模块1 │ 模块2 │ 模块3 │ 模块4 │ 模块5 │   共享服务      │
│ 数据  │ 选题  │ 视频  │ 发布  │ 监控  │               │
│ 采集  │ 策略  │ 生成  │ 分发  │ 迭代  │ · 素材库      │
│      │      │      │      │      │ · 脚本库      │
│      │      │      │      │      │ · 对标账号库  │
│      │      │      │      │      │ · 内容资产库  │
│      │      │      │      │      │ · 人设配置    │
└──────┴──────┴──────┴──────┴──────┴──────────────────┘
```

**设计原则**：
- **统一主干**：一个任务贯穿5个模块，数据自动流转，不需要手动搬运
- **可插拔模块**：每个模块可独立运行、独立升级，换工具不影响其他模块
- **功能优先**：外部工具更好就用外部工具，能做成更好的就替换
- **短期单人，长期团队**：先满足单人使用，架构预留多人协作空间

---

## 全局字段约定

- **大数值格式**：≥10000 显示为"X.X万"，如 12345 → 1.2万
- **百分比精度**：保留1位小数，如 3.2%
- **互动率公式**：(点赞+评论+收藏)/播放量 × 100%，播放量为0时显示"--"
- **时间格式**：视频时长用 MM:SS（如 03:25），日期用 MM-DD HH:mm
- **金额格式**：¥ 前缀，无小数

## 用户故事

### 核心故事（5个）

1. 作为运营者，我想要每天自动抓取对标账号的爆款视频数据并提取文案，以便快速了解竞品动态
2. 作为运营者，我想要基于爆款数据分析和当日热点，生成当日选题和脚本骨架，以便内容决策有据可依
3. 作为运营者，我想要把文案转化为包含口播、地图动画、音乐、字幕的多频共振视频，以便高效产出高质量内容
4. 作为运营者，我想要自动生成多平台（小红书/抖音/视频号）的标题、描述和话题标签，以便快速发布
5. 作为运营者，我想要监控发布后数据并自动分析，把结论翻译成明天的创作改进指令，以便持续进化

### 重要故事（5个）

6. 作为运营者，我想要把口播文案翻译成带导演思维的分镜脚本（含视觉/听觉/特效/时间轴），以便精确控制视频节奏
7. 作为运营者，我想要实现"一鱼多吃"——一条视频素材自动派生为多平台不同形态的内容
8. 作为运营者，我想要统一管理素材、脚本、对标账号数据等所有内容资产，以便随时复用
9. 作为运营者，我想要系统内置"去AI味"规则，确保生成文案符合我的语言风格
10. 作为运营者，我想要系统自动追踪当日热点并推荐与选题的结合方式，以便借势流量

### 锦上添花（3个）

11. 作为运营者，我想要自动完成GEO优化，让内容被AI搜索引擎引用
12. 作为运营者，我想要搭建全自动的监控工作流（n8n），实现无人值守
13. 作为运营者，我想要一键生成结构化图文攻略，用于小红书图文发布和GEO布局

## 逐页功能说明

### 页面：仪表盘（任务中心）

**功能概述**：系统的指挥中心，展示当日任务状态、待办事项、最新数据摘要，统一调度各模块

**页面分区**：

| 区块名称 | 功能说明 | 包含元素 |
|---------|---------|---------|
| 顶部状态栏 | 展示当前工作流进度 | 步骤指示器：数据采集 → 选题 → 视频 → 发布 → 监控 |
| 今日任务卡片 | 展示当日待办和进行中的任务 | 任务卡片列表，状态标签（待办/进行中/已完成） |
| 数据速览 | 关键数据摘要 | 昨日发布视频数据、对标账号最新爆款、待确认事项数 |
| 快捷操作 | 一键进入各模块 | 模块入口按钮 |
| 热点推荐 | 当日热点信息流 | 热点卡片列表，含匹配度评分 |

**显示字段**：

| 字段 | 显示方式 | 数据来源 | 备注 |
|------|---------|---------|------|
| 任务标题 | 文字 | 用户创建/系统生成 | 如"北疆7日环线视频" |
| 任务状态 | 状态标签 | 系统跟踪 | 待办/采集/选题/制作/发布/监控/已完成 |
| 当前步骤 | 高亮步骤指示器 | 系统跟踪 | 标识任务在5步流程中的位置 |
| 爆款视频数 | 数字 | 模块1产出 | 今日新增对标爆款数 |
| 昨日数据 | 数据摘要 | 模块5产出 | 播放量/互动率/完播率 |

**操作定义**：

| 操作 | 触发方式 | 行为 | 用户反馈 | 权限/条件 |
|------|---------|------|---------|----------|
| 创建新任务 | 点击"新建任务"按钮 | 进入任务创建流程，填写选题方向 | 成功：Toast提示"任务已创建"+跳转任务详情；失败：错误提示 | 所有 |
| 进入模块 | 点击模块入口卡片 | 跳转对应模块页面 | 即时跳转 | 所有 |
| 查看任务详情 | 点击任务卡片 | 展开任务完整信息，可跳转到当前步骤 | 展开详情面板 | 所有 |
| 热点结合 | 点击热点卡片 | 跳转选题工作台，自动带入热点信息 | 即时跳转+Toast提示"已带入热点信息" | 所有 |

**页面状态**：

| 状态 | 表现 |
|------|------|
| 无任务 | 显示引导卡片："创建你的第一个任务" |
| 任务进行中 | 高亮当前步骤，显示进度百分比 |

**业务规则**：
- 每天默认创建一个"当日内容任务"
- 任务状态由模块执行自动推进，也可手动回退
- 任务数据在各模块间自动流转，不需要手动复制粘贴

| 操作 | 触发方式 | 行为 | 用户反馈 | 权限/条件 |
|------|---------|------|---------|----------|
| 新建采集任务 | 点击"新建任务" | 配置采集参数并提交 | 成功：Toast提示"任务已创建"+关闭弹窗；失败：错误提示 | 所有 |

- 采集频率默认每日一次，可调整为每6小时/每12小时/每日
- 单次采集视频数上限100条，超过自动分批
- 文案提取在视频下载后自动触发，使用 ffmpeg + faster-whisper
- 采集失败自动重试1次，仍失败则标记为"待手动处理"

**显示字段**：

| 字段 | 显示方式 | 数据来源 | 备注 |
|------|---------|---------|------|

| 操作 | 触发方式 | 行为 | 用户反馈 | 权限/条件 |
|------|---------|------|---------|----------|

| 删除口头禅 | 点击删除 | 删除该口头禅 | 弹出二次确认弹窗→确认后Toast提示"已删除" | 所有 |
| 上传样稿 | 点击上传 | 上传文本或音频文件 | 成功：Toast提示"上传成功"；失败：错误提示 | 所有 |
| 预览效果 | 点击"预览" | AI用当前人设配置生成一段示例文案，展示去AI味效果 | Loading旋转+结果替换展示 | 所有 |
| 保存配置 | 点击"保存" | 保存人设配置，同步更新到脚本编辑器的去AI味引擎 | 成功：Toast提示"配置已保存"；失败：错误提示 | 所有 |

**业务规则**：

- 至少填写"称呼"和"一句话定位"才能保存
- 口头禅在文案生成时自动应用，不强制但优先匹配
- 样稿为可选项，有样稿时AI风格学习效果更好

### 热点+爆款文案融合流程

热点追踪和爆款分析不是两条平行线，而是在选题工作台中**汇合**。融合流程如下：

```
                    ┌── 热点追踪 ──筛选匹配热点──┐
                    │                             │
用户创建任务 ──→ 选题工作台 ←── 爆款视频库 ──选定对标视频
                    │                             │
                    └── AI融合分析 ──生成选题方案 ──┘
```

**融合步骤**：

1. **输入**：1-5条爆款视频（含文案+数据）+ 1-3条匹配热点
2. **AI融合分析**：分析爆款结构 + 热点情绪 + 用户人设 → 生成3个选题方案
3. **每个选题方案必须包含**：
   - 对标来源（参考了哪些爆款视频的结构）
   - 热点连接方式（引子/佐证/解决方案，三种方式之一）
   - 热点有效期提醒（是否在24-48小时窗口内）
4. **用户选择**：选用/融合/重新生成
5. **输出**：确认的选题 + 脚本骨架 → 进入脚本编辑器

**热点连接的三种方式**（原文提炼）：

| 方式 | 逻辑 | 示例 |
|------|------|------|
| 热点作引子 | 热点事件→用户痛点→引出你的方案 | "独库公路开通冲上热搜，很多人问会不会又堵？别急，我帮你规划一条绕开堵点的秘境路线…" |
| 热点作佐证 | 热点观点→认同/反对→引出你的逻辑 | "特种兵旅游被官媒批评走马观花，深有同感。在新疆，慢才是奢侈…" |
| 热点作解决方案 | 热点需求→给出替代方案→展示专业 | "高考结束，收到好多私信问毕业旅行。预算有限？走伊犁小环线，5天草原湖泊雪山全有…" |

**去AI味规则（内置检查引擎）**：

*绝对禁止词句（触发时强制标红提示替换）*：
- "总的来说""总而言之""此外""与此同时""不可错过""必打卡""不容错过"
- "让我们一起""相信大家""众所周知""不言而喻"
- 超过25个字的长句（一口气读不完的都算）

*强制执行规则（生成时自动应用）*：
- **观点要极端**：不许端水，必须有明确偏好。如"整个独库公路，我只认北段。南段？浪费时间"
- **用身体感受代替数据**：不说"海拔3200米含氧量低"，说"下车刚走两步，脚底跟踩棉花似的"
- **暴露职业强迫症**：每篇至少一处只有从业者才注意的细节吐槽
- **口语化替换**："因此"→"所以呢"、"体验绝佳"→"真叫一个爽"
- **每句话≤20字**，超长句自动拆分

*人设注入（从人设配置页读取）*：
- 自动应用用户设定的语言风格（短句/打比方/反问/吐槽）
- 自动应用固定口头禅（开场白、转折语、揭晓语）
- 统一使用第一人称叙事视角

*AI口播音频生成*：
- 工具：edge-tts（免费），默认中文普通话，可选音色列表
- 输入：口播文本全文
- 输出：MP3音频文件，自动关联到对应分镜段落
- 预览：生成后可在线播放，不满意可重新生成或手动替换
- 进阶：后续可切换为 ElevenLabs（声音克隆，需付费）

**业务规则**：
- 口播占比检查：低于设定阈值时警告
- 素材需求自动关联素材库，标记已有/需新增
- 去AI味检查不通过时，禁止进入视频生成步骤

**去AI味检查标准**：

| 级别 | 条件 | 处理 |
|------|------|------|
| ✅ 通过 | 0个违规项 | 允许进入视频生成 |
| ⚠️ 警告 | 1-2个建议级问题 | 允许进入视频生成，显示改进建议 |
| ❌ 不通过 | 1个以上禁止级问题 | 禁止进入视频生成，必须修改 |

**检查项**：
- 绝对禁止词（"众所周知"、"不言而喻"、"毫无疑问"等AI高频词）
- 超长单句（>30字）
- 缺乏口语化表达的段落
- 重复句式结构

| 标记元素亮点 | 点击"标记亮点" | 为某个元素打上正面标签 | 即时标记+Toast提示"已标记" | 所有 |
| 导出资产 | 点击导出 | 下载脚本/数据为文件 | Loading+浏览器下载 | 所有 |
| 删除资产 | 点击删除 | 确认后删除 | 弹出二次确认弹窗→确认后Toast提示"已删除" | 仅草稿状态 |

**业务规则**：
- AI自动分析每条发布视频，标记"高完播""高互动""高转化"等亮点标签
- 素材复用时自动检查版权状态，已过期的版权素材提醒重新确认
- 发布30天后的视频自动归档，归档内容可搜索但不在默认列表展示

## 边缘场景

### 数据采集（模块1）

**正常流**：输入对标账号链接 → 自动采集视频列表 → 提取文案 → 入库

**异常流**：
- 平台反爬限制 → 降低频率 / 切换采集方案 / 提示手动补充
- 视频号采集失败 → 提示视频号暂不支持自动采集，建议手动记录
- 文案提取失败（口音/噪音） → 提示人工校对，标记为"待确认"
- 账号已注销或改名 → 标记异常，提醒用户核实

**边界值**：
- 对标账号数=0 → 提示先添加对标账号
- 单次采集视频数过多（>100） → 分批采集，防止封禁
- 视频无口播（纯音乐风景） → 标记为"无文案"，不进入文案分析

### 选题分析（模块2）

**正常流**：选定对标视频 → AI分析爆款结构 → 生成3个选题 → 用户选择 → 生成脚本骨架

**异常流**：
- 选定视频不足3条 → 提示添加更多对标素材
- AI生成的选题与热点不匹配 → 提示用户手动调整或换一批
- 文案提取质量差（AI味重） → 触发"去AI味"规则重写

### 视频生成（模块3）

**正常流**：分镜脚本 → 自动组装素材 → 预览 → 调整 → 渲染MP4

**异常流**：
- 素材缺失 → 高亮缺失素材，链接到素材库添加
- 渲染失败（内存/编码） → 提示具体错误，建议分段渲染
- AI口播生成失败 → 提示手动录制或更换语音服务
- 视频时长超出限制 → 提示拆分或裁剪

### 发布（模块4）

**正常流**：选择视频 → 生成多平台文案 → 审核 → 手动发布 → 标记状态

**异常流**：
- 平台审核不通过 → 提示违规原因，建议修改
- 话题标签被平台判定无效 → 提示更换标签

### 数据监控（模块5）

**正常流**：录入数据 → AI分析 → 生成改进建议 → 应用到下期任务

**异常流**：
- 数据不足无法分析 → 提示积累更多数据后再分析
- 改进建议与上期重复 → 提示"该问题已连续出现，建议考虑更大调整"

### 并发场景

| 场景 | 处理方式 |
|------|---------|
| 同一脚本在两个标签页同时编辑 | 乐观锁，保存时检测版本冲突，提示"内容已被修改，请刷新后重试" |
| 多个采集任务并行执行 | 队列化处理，单账号间隔≥5秒，避免触发平台反爬 |
| AI生成选题时用户同时修改人设 | 中断当前生成，提示"人设已变更，是否使用新人设重新生成？" |
| 同一视频同时加入多个选题 | 允许复用，各选题独立管理引用关系 |

### 全局异常处理

| 异常类型 | 处理方式 |
|---------|---------|
| AI服务超时（>30秒） | Toast提示"AI服务响应超时，请稍后重试"，保留已填写的表单数据 |
| AI服务不可用 | 操作按钮禁用+Tooltip"AI服务暂不可用"，非AI功能不受影响 |
| 数据库连接失败 | 全页错误状态"系统连接异常"，提供重试按钮 |
| 磁盘空间不足（渲染/上传时） | 提前检查可用空间，不足时警告"磁盘空间不足，请清理后重试" |
| 平台API频率限制 | 采集任务自动降速（增加间隔），显示"已触发平台限制，自动降速采集中" |

## 合规底线

1. 素材只用免费可商用来源或AI生成，不碰侵权素材
2. AI生成内容发布时主动标识，不隐藏平台添加的AI标识
3. 数据采集遵守平台规则，控制频率，不进行大规模商业数据贩卖
4. 评论区互动用真实账号，禁止虚假评论和水军
5. 不碰敏感、有争议的社会公共事件作为热点
