## 6. 状态设计

### 6.1 SEO检查状态

| 状态 | 评分颜色 | 标记已发布按钮 |
|------|---------|-------------|
| >=80 优秀 | $status-success | 可用 |
| 60-79 良好 | $status-success | 可用 |
| 40-59 需改进 | $status-warning | 可用, 显示警告 |
| <40 不合格 | $status-error | 禁用, 显示"SEO评分过低" |

### 6.2 AI文案生成状态

| 状态 | 表现 |
|------|------|
| 空闲 | 标题/描述输入框有占位符提示 |
| 生成中 | 输入框显示骨架屏 + "AI 正在生成..." |
| 生成完成 | 输入框填充生成内容, 用户可编辑 |
| 换一批 | 清空当前内容, 重新生成 |

### 6.3 平台文案切换状态

| 状态 | 表现 |
|------|------|
| 切换平台 | 淡入淡出过渡, 300ms |
| 各平台文案独立 | 切换时加载对应平台已保存的文案 |
| 未生成文案 | 输入框显示"点击生成"按钮 |
| 文案已修改 | 标题行显示"已修改"标记 |

### 6.4 发布状态流转

| 状态 | 指示色 | 操作 |
|------|--------|------|
| 未发布 | $text-tertiary "⬜" | 可点击"标记已发布" |
| 待发布 | $status-info "⏳" | 手动更新 |
| 已发布 | $status-success "✅" | 显示发布时间, 可撤回 |
| 发布失败 | $status-error "✗" | 可重试 |

### 6.5 加载与空状态

| 状态 | 表现 |
|------|------|
| 首次加载 | 双栏显示骨架屏 |
| 无视频 | 左栏显示"请先完成视频渲染"引导 |
| SEO未检查 | 评分区显示"点击检查获取SEO评分" |
| 评论区无建议 | "生成视频后AI将自动生成运营建议" |
| 网络错误 | 红色提示条 + "加载失败，点击重试" |
