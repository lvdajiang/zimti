> 版本: v1.0 | 日期: 2026-05-10
> 模式: 微服务（FastAPI 独立服务）

## 整体架构

```
┌──────────────────────────────────────────────────────┐
│                   Node.js 主后端                       │
│            (Express/Koa + TypeScript)                 │
│                                                      │
│  /api/v1/collect-tasks     /api/v1/viral-videos      │

└──────┬──────────┬──────────┬──────────┬──────────────┘
       │          │          │          │
       │ HTTP     │ HTTP     │ HTTP     │ child_process
       ▼          ▼          ▼          ▼
┌────────────┐ ┌────────┐ ┌──────┐ ┌──────────┐
│ Crawler    │ │Whisper │ │ TTS  │ │  ffmpeg  │
│ Service    │ │Service │ │Service│ │ (CLI直调) │
│ :8001      │ │:8002   │ │:8003  │ │ 系统安装   │
│            │ │        │ │       │ │          │
│ MediaCrawler│ │faster- │ │edge- │ │ 视频裁剪  │
│ 抖音采集    │ │whisper │ │tts   │ │ 音频提取  │
│ 小红书采集  │ │语音转文│ │语音合成│ │ 格式转换  │
└────────────┘ └────────┘ └──────┘ └──────────┘
```

## 为什么 ffmpeg 用 CLI 直调？

| 工具 | 特点 | 集成方式 |
|------|------|---------|
| MediaCrawler | Python爬虫框架，复杂状态管理 | 微服务（需要独立进程管理Cookie/反爬） |
| faster-whisper | Python + CUDA模型，重计算 | 微服务（需要GPU资源管理） |
| edge-tts | Python异步库，轻量 | 微服务（统一Python服务管理） |
| ffmpeg | 系统二进制，命令行工具 | **CLI直调**（无需额外服务开销） |

ffmpeg 是无状态 CLI 工具，Node.js 直接 spawn 调用最高效，不需要包装成服务。

## 错误处理策略

| 场景 | 处理 |
|------|------|
| 微服务不可用 | Node.js 捕获连接错误，返回 503 + Toast "AI服务暂不可用" |
| 采集触发反爬 | Crawler Service 返回 429，Node.js 标记任务"已限流"，自动降速重试 |
| 转录超时(>5min) | Whisper Service 超时返回，Node.js 标记"转录失败"，支持手动重试 |
| TTS 生成失败 | TTS Service 返回错误详情，Node.js 提示"语音生成失败，请重试" |
| ffmpeg 进程崩溃 | 捕获 exit code != 0，记录 stderr，标记渲染失败 |

## 重试规则

| 操作 | 重试次数 | 间隔 | 条件 |
|------|---------|------|------|
| 视频采集 | 1次 | 30秒 | 反爬限制(429) |
| 文案提取 | 2次 | 60秒 | 转录失败 |
| TTS生成 | 1次 | 10秒 | 网络错误 |
| ffmpeg | 0次 | — | 不重试（参数错误需人工排查） |

## 服务5：Remotion 渲染引擎（后端渲染 + Vue 自建预览）

### 核心思路

```
Vue 前端（自建预览）          Node.js 后端（Remotion 渲染）
┌──────────────────┐         ┌──────────────────────┐
│ HTML5 <video>     │         │ @remotion/renderer   │
│ 自建时间轴        │ ──────→ │ @remotion/bundler    │
│ 素材拖拽面板      │  JSON   │ React Composition    │
│ 参数调整面板      │  配置    │                      │
│                   │ ←────── │ 渲染为 MP4           │
│ 实时预览用低分辨率 │  MP4    │ 多版本派生           │
└──────────────────┘         └──────────────────────┘
```

**关键决策**：Remotion 只在后端运行，前端不引入 React。Vue 用 HTML5 video + Canvas 自建预览体验。

### 为什么不用 @remotion/player？

| 方案 | 优点 | 缺点 |
|------|------|------|
| @remotion/player（React） | 精确到帧的预览、Composition实时渲染 | 需要在Vue中引入React，构建复杂，包体积大 |
| **Vue + HTML5 video**（选择） | 纯Vue生态、构建简单、无React依赖 | 预览精度为秒级（非帧级），需自建时间轴 |

对于口播+素材拼合类视频，秒级预览精度完全够用。帧级精度仅在最终渲染时需要（由Remotion后端保证）。

### 后端 Remotion 集成

#### 目录结构

```
server/
├── src/
│   └── services/
│       └── remotion/
│           ├── renderer.ts        # 渲染服务封装
│           └── compositions/      # Remotion Composition（React）
│               ├── Root.tsx       # 入口：注册所有Composition
│               ├── MainVideo.tsx  # 主版（1-3分钟小红书）
│               ├── DouyinClip.tsx # 抖音版（30-60秒快剪）
│               ├── WeixinVideo.tsx# 视频号版（1-2分钟利他型）
│               └── shared/        # 共享组件
│               ├── Subtitle.tsx
│               ├── BackgroundMusic.tsx
│               ├── MapAnimation.tsx
│               └── TransitionEffect.tsx
```

#### 渲染 API

```typescript
// services/remotion/renderer.ts
import { bundle } from '@remotion/bundler'
import { renderMedia, getCompositions } from '@remotion/renderer'

export class RemotionRenderer {
  /** 渲染视频 */
  async render(config: RenderConfig): Promise<RenderResult>

  /** 渲染多版本（一鱼多吃） */
  async renderMultiVersion(parentVideoId: string): Promise<MultiVersionResult>

  /** 获取Composition列表 */
  async listCompositions(): Promise<Composition[]>

```

#### RenderConfig 接口

```typescript
interface RenderConfig {
  composition_id: 'MainVideo' | 'DouyinClip' | 'WeixinVideo'
  // 分镜数据
  segments: {
    id: string
    duration: number            // 秒
    oral_audio_url: string      // 口播音频
    oral_text: string           // 口播文字（用于字幕）
    background_music_url?: string
    background_music_volume: number  // 0-1
    visual_type: 'video' | 'image' | 'map_animation' | 'solid_color'
    visual_source: string       // 素材URL
    subtitle_style: {
      font_size: number
      color: string
      position: 'bottom' | 'center' | 'top'
      background_opacity: number
    }
    transition?: {
      type: 'fade' | 'slide_left' | 'slide_right' | 'zoom' | 'none'
      duration: number          // 毫秒
    }
  }[]
  // 全局配置
  width: number                 // 主版1080，抖音1080，视频号1080
  height: number                // 主版1920，抖音1920，视频号1920
  fps: number                   // 30
  output_format: 'mp4' | 'webm'
  output_path: string
}
```

### 前端 Vue 预览器

#### 组件结构

```
VideoPreview.vue
├── PreviewPlayer.vue        # HTML5 video 播放器
│   ├── video 元素            # 播放拼接后的低分辨率预览视频
│   └── 自定义控制栏          # 播放/暂停/进度条/音量

├── AdjustPanel.vue          # 参数调整面板（字幕样式/音量/转场）
└── RenderControl.vue        # 渲染操作栏（预览/渲染/多版本）
```

#### 预览策略

| 操作 | 实现方式 |
|------|---------|
| 实时预览 | 后端根据当前素材配置，生成低分辨率（480p）拼接预览视频，前端用 `<video>` 播放 |
| 素材替换 | 修改配置后，请求后端重新生成预览片段（只重新生成被修改的分段） |
| 时间轴拖拽 | 纯前端操作，修改 segments 的 duration 顺序，拖拽结束后提交后端刷新预览 |
| 最终渲染 | 确认后提交完整 RenderConfig，后端用 Remotion 渲染全分辨率（1080p）MP4 |

#### 预览优化

```
初次加载 → 后端生成 480p 全片段预览视频（10-30秒）
素材替换 → 只重新渲染被修改的分段，拼接回预览视频（3-5秒）
参数调整 → 字幕样式/音量调整不需要重新渲染（前端CSS叠加）
最终渲染 → Remotion 渲染 1080p 完整视频（1-3分钟）
```

### "一鱼多吃"版本规则

| 版本 | Remotion Composition | 时长 | 调整逻辑 |
|------|---------------------|------|---------|
| 主版（小红书） | MainVideo.tsx | 1-3min | 完整分镜、70%口播、封面图文并茂 |
| 抖音版 | DouyinClip.tsx | 30-60s | 前3秒钩子强化、快节奏裁剪、加热门BGM、自动提取高光片段 |
| 视频号版 | WeixinVideo.tsx | 1-2min | 利他型标题叠加、开场加"给想去XX的朋友"、结尾社交互动引导 |
| 图文版 | — | — | 不经过Remotion，后端直接从口播文案生成图文攻略HTML |

## 服务6：地图3D路线动画服务

### 核心思路

```
用户输入路线（GPX/坐标） → 地图动画服务 → 生成3D飞越视频片段 → 存入素材库 → Remotion合成到最终视频
```

Vue前端提供路线编辑器（选点、拖拽路线），后端渲染服务用MapLibre GL + Three.js在无头浏览器中生成3D路线飞行动画。

### 技术选型

| 组件 | 技术 | 用途 |
|------|------|------|
| 地图底图 | OpenStreetMap瓦片（免费） | 道路、地名、卫星图 |
| 地图渲染 | MapLibre GL JS（开源） | 2D/3D地图引擎 |
| 3D效果 | Three.js | 相机飞行、高度拉伸、地形 |
| 路线计算 | Turf.js | 路线距离、中心点、边界框 |
| 帧捕获 | Puppeteer（无头Chrome） | 逐帧截图 → ffmpeg合成视频 |
| 路线数据来源 | 奥维地图导出GPX / 用户手动标点 | 路线坐标输入 |

### 为什么这样选？

| 方案 | 优点 | 缺点 |
|------|------|------|
| 奥维地图手动录屏 | 零开发 | 全手动、分辨率受限、不可复用 |
| Mapbox GL 3D | 效果精美、API丰富 | 商用收费（>50k次/月收费） |
| **MapLibre GL + Three.js**（选择） | 开源免费、可编程控制、效果够用 | 需要自建3D相机动画逻辑 |

### 动画效果类型

提供3种预设动画模板：

| 模板 | 效果 | 参数 | 适用场景 |
|------|------|------|---------|
| **鸟瞰飞越** | 相机高空沿路线飞行，俯瞰全貌 | 高度: 500-2000m, 俯角: 45-60°, 速度: 可调 | 环线总览、路线概览 |
| **沿线巡航** | 相机贴近地面，沿路线缓慢推进 | 高度: 50-200m, 俯角: 15-30°, 速度: 可调 | 路段展示、景点过渡 |
| **3D地标标记** | 2D地图上弹出3D标记点，连线动画 | 标记高度: 50-100px, 连线颜色: 可配置 | 多目的地展示、行程概览 |

### API 契约

#### 生成地图动画（异步）

```

```

### 渲染流程

```
1. 用户在Vue路线编辑器中画路线、设参数
2. 提交JSON配置到后端
3. 后端启动Puppeteer无头浏览器
4. 加载MapLibre GL + Three.js渲染页面
5. 按相机路径逐帧截图（1920x1080 @30fps）
6. ffmpeg将帧序列合成MP4
7. 返回视频URL，自动存入素材库
8. 用户在视频预览页将该片段拖入时间轴
```

### 地图瓦片源

| 来源 | 用途 | 许可 |
|------|------|------|
| OpenStreetMap | 道路/地名底图 | ODbL（免费，需标注来源） |
| Maptiler卫星图 | 卫星影像底图 | 免费额度（每月10万次） |
| ASTER GDEM | 地形高程数据（可选3D地形） | 免费（NASA） |

### 与 Remotion 的关系

地图动画服务**独立于 Remotion**：
- 地图动画由 Puppeteer + MapLibre 渲染，输出为独立 MP4 文件
- 生成的 MP4 存入素材库，作为普通视频素材使用
- Remotion 在最终合成时引用该素材文件
- 两套渲染引擎互不依赖
