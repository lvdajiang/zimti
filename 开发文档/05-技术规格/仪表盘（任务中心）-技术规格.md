## 2. 组件 Props 定义

### WorkflowStepper.vue

| Prop | 类型 | 必填 | 说明 | 来源字段 |
|------|------|------|------|---------|
| current_step | `number` | 是 | 当前工作流步骤编号(1-5) | 📊计算 dashboardStore.current_workflow_step |
| steps | `WorkflowStep[]` | 是 | 步骤定义数组 | ⛔禁止 硬编码常量 |

### WorkflowStep.vue

| Prop | 类型 | 必填 | 说明 | 来源字段 |
|------|------|------|------|---------|
| step | `WorkflowStepItem` | 是 | 步骤数据 | 📥存储/📊计算 |

### TaskCard.vue

| Prop | 类型 | 必填 | 说明 | 来源字段 |
|------|------|------|------|---------|
| task | `DashboardTask` | 是 | 任务数据对象 | 📥存储 tasks 表 |

### 数据类型定义（TypeScript interface）

```typescript
// 工作流步骤定义
interface WorkflowStep {
  name: string       // ⛔禁止 硬编码步骤名
  index: number      // ⛔禁止 步骤序号 1-5
}

interface WorkflowStepItem {
  name: string
  index: number
  status: 'completed' | 'current' | 'pending'  // 📊计算 对比 current_workflow_step
}

// 工作流API响应
interface WorkflowResponse {
  current_step: number    // 📊计算 MAX(tasks.current_step)
  steps: WorkflowStepItem[]
}

// 今日任务
interface DashboardTask {
  id: string                       // 📥存储 tasks.id
  title: string                    // 📥存储 tasks.title
  status: TaskStatus               // 📥存储 tasks.status
  current_step: string             // 📥存储 tasks.current_step
  progress_percent: number         // 📊计算 后端计算百分比
}

type TaskStatus = 'todo' | 'collecting' | 'selecting' | 'producing' | 'publishing' | 'monitoring' | 'completed'

type TaskFilter = 'all' | 'todo' | 'in_progress' | 'done'

// 任务列表API响应
interface TaskListResponse {
  items: DashboardTask[]
  total: number                    // 📊计算 总数
}

// 数据速览
interface DashboardStats {
  yesterday_play_count: number     // 📊计算 SUM(video_metrics.play_count)
  play_count_trend: number | null  // 📊计算 环比趋势百分比
  new_viral_count: number          // 📊计算 COUNT(viral_videos)当日
  viral_trend: number | null       // 📊计算 差值
  pending_items_count: number      // 📊计算 COUNT(pending_items)
  pending_description: string      // 📊计算 动态文案
  interaction_rate: number | null   // 📊计算 互动率百分比
  interaction_trend: number | null  // 📊计算 差值
}

// 热点推荐
interface HotspotItem {
  id: string                       // 📥存储 hotspots.id
  title: string                    // 📥存储 hotspots.title
  source_platform: string          // 📥存储 hotspots.source_platform
  source: string                   // 📥存储 hotspots.source
  relevance_score: number          // 📊计算 AI匹配度百分比
}

interface HotspotListResponse {
  items: HotspotItem[]
}

// 用户信息
interface UserInfo {
  id: string                       // 📥存储 users.id
  username: string                 // 📥存储 users.username
  avatar_url: string | null        // 📥存储 users.avatar_url
}

// 通知计数
interface UnreadCountResponse {
  count: number                    // 📊计算 COUNT(notifications WHERE is_read=false)
}

// 创建任务请求
interface CreateTaskRequest {
  title: string                    // 📥存储 表单输入
  description?: string             // 📥存储 表单输入
}

// 创建任务响应
interface CreateTaskResponse {
  id: string                       // 📥存储 新任务ID
  title: string
  status: TaskStatus
  current_step: string
  created_at: string               // 📥存储 创建时间
}
```

## 5. 状态管理

| 数据 | 归属（组件ref/Pinia/URL） | 说明 |
|------|--------------------------|------|
| 工作流状态 (current_workflow_step, steps[]) | Pinia: `dashboardStore` | 页面挂载时 GET /dashboard/workflow；页面可见性变化/任务变更时刷新 |
| 今日任务列表 (tasks[]) | Pinia: `dashboardStore` | 页面挂载时 GET /tasks?date=today；筛选切换、创建任务后、定时轮询5min刷新 |
| 任务筛选值 (filter) | 组件ref | 默认 "all"；用户点击筛选标签切换 |
| 数据速览 (stats) | Pinia: `dashboardStore` | 页面挂载时 GET /dashboard/stats；定时轮询5min刷新 |
| 热点推荐 (hotspots[]) | Pinia: `dashboardStore` | 页面挂载时 GET /hotspots/recommended；定时轮询10min；WebSocket预留推送 |
| 未读通知数 (unread_count) | Pinia: `notificationStore` | App初始化时 GET /notifications/unread-count；WebSocket推送/阅读后刷新 |
| 当前用户信息 (user) | Pinia: `userStore` | App初始化时 GET /users/me；登录/登出时刷新 |
| 当前日期 (current_date) | 组件ref | 页面挂载时 new Date()；跨日定时器更新 |
| 新建弹窗可见 (createModalVisible) | 组件ref | 默认 false；点击"新建任务"按钮切换 |

### 加载策略

1. **页面挂载**：并行发起 5 个请求（workflow, tasks, stats, hotspots, unread_count），各区块独立 loading 状态
2. **骨架屏**：各区块显示对应数量的骨架卡片，pulse 1.5s 动画
3. **定时刷新**：
   - 任务列表：5分钟轮询
   - 数据速览：5分钟轮询
   - 热点推荐：10分钟轮询
4. **增量更新**：任务列表顶部 2px 线性进度条（$brand-indigo）
5. **WebSocket 预留**：热点推送、通知计数实时更新通道
