### 数据类型定义（TypeScript interface）

```typescript
// 采集任务状态
type CollectTaskStatus = 'waiting' | 'running' | 'paused' | 'completed' | 'failed'

// 采集类型
type CollectType = 'video' | 'transcript' | 'hotspot'

// 类型筛选
type TypeFilter = 'all' | 'video' | 'transcript' | 'hotspot'

// 采集任务
interface CollectTask {
  id: string                       // 📥存储 collect_tasks.id
  task_id: string                  // 📥存储 collect_tasks.task_id（编号如"DC-001"）
  task_type: CollectType           // 📥存储 collect_tasks.task_type
  target_description: string      // 📥存储 collect_tasks.target_description
  target_account_id: string       // 🔗调用 collect_tasks.target_account_id → benchmark_accounts.id
  target_account_name: string     // 🔗调用 benchmark_accounts.account_name（JOIN获取）
  status: CollectTaskStatus       // 📥存储 collect_tasks.status
  progress: number                // 📥存储 collect_tasks.progress（已处理数量）
  total_count: number             // 📥存储 collect_tasks.total_count
  success_count: number           // 📥存储 collect_tasks.success_count
  fail_count: number              // 📥存储 collect_tasks.fail_count
  progress_percent: number        // 📊计算 progress/total_count*100
  success_fail_display: string    // 📊计算 "{success_count}/{total_count}"
  duration: string                // 📊计算 后端格式化耗时
  created_at: string              // 📥存储 collect_tasks.created_at
}

// 采集任务统计
interface CollectStats {
  today_total: number             // 📊计算 COUNT(collect_tasks) WHERE created_at >= today_start
  running_count: number           // 📊计算 COUNT WHERE status='running'
  completed_count: number         // 📊计算 COUNT WHERE status='completed'
  failed_count: number            // 📊计算 COUNT WHERE status='failed'
}

// 任务列表API响应
interface CollectTaskListResponse {
  items: CollectTask[]
  total: number                   // 📊计算 总数
}

// 统计摘要API响应
interface CollectStatsResponse {
  today_total: number             // 📊计算
  running_count: number           // 📊计算
  completed_count: number         // 📊计算
  failed_count: number            // 📊计算
}

// 新建/编辑任务请求
interface CollectTaskFormRequest {
  task_type: CollectType          // 📥存储 表单下拉
  target_account_id: string       // 🔗调用 从对标账号列表选择
  date_range?: { start: string; end: string }  // 📥存储 表单日期选择器
  max_count: number               // 📥存储 表单数字输入，默认100
}

// 任务表单数据（含初始值）
interface CollectTaskFormData {
  task_type: CollectType
  target_account_id: string
  date_range?: { start: string; end: string }
  max_count: number
}

// 目标账号下拉选项
interface AccountOption {
  id: string                      // 📥存储 benchmark_accounts.id
  account_name: string            // 📥存储 benchmark_accounts.account_name
  platform: string                // 📥存储 benchmark_accounts.platform
}

// 日志条目
interface LogEntry {
  timestamp: string               // 📥存储 collect_task_logs.timestamp
  level: 'success' | 'error' | 'info'  // 📥存储 collect_task_logs.level
  message: string                 // 📥存储 collect_task_logs.message
}

// 日志响应
interface LogResponse {
  entries: LogEntry[]
  has_more: boolean               // 📊计算 是否有更多日志
}

// 定时配置
interface ScheduleConfig {
  frequency: '6h' | '12h' | 'daily'  // 📥存储 schedule_configs.frequency
  execute_time: string            // 📥存储 schedule_configs.execute_time
  is_enabled: boolean             // 📥存储 schedule_configs.is_enabled
}

// 删除响应
interface DeleteResponse {
  success: boolean
}

// 批量删除请求
interface BatchDeleteRequest {
  ids: string[]                   // 📥存储 选中的任务ID数组
}

// 批量删除响应
interface BatchDeleteResponse {
  success: boolean
  deleted_count: number           // 📊计算
}
```

## 5. 状态管理

| 数据 | 归属（组件ref/Pinia/URL） | 说明 |
|------|--------------------------|------|
| 任务列表 (tasks[]) | Pinia: `collectTaskStore` | 页面挂载时 GET /collect-tasks；类型筛选切换、搜索(防抖500ms)、增删改操作后刷新 |
| 类型筛选值 (type_filter) | URL参数 + 组件ref | 默认 "all"；用户点击筛选标签切换 |
| 搜索关键词 (search_keyword) | 组件ref | 默认 ""；防抖500ms后触发列表刷新 |
| 统计摘要 (stats) | Pinia: `collectTaskStore` | 页面挂载时 GET /collect-tasks/stats；新建/执行/暂停/删除/重试后刷新 |
| 分页 (page, page_size, total) | URL参数 + 组件ref | page=1, page_size=20；翻页/切换每页条数 |
| 已选行 (selected_ids) | 组件ref | 默认 []；复选框勾选/取消 |
| 新建弹窗可见 (createModalVisible) | 组件ref | 默认 false；点击"新建任务"按钮 |
| 编辑模式 (editingTask) | 组件ref | null(新建) / task对象(编辑)；仅等待中状态可编辑 |
| 日志弹窗可见 (logModalVisible) | 组件ref | 默认 false；点击"查看日志"按钮 |
| 当前查看日志的任务ID (loggingTaskId) | 组件ref | null；点击"查看日志"时设置 |
| 日志条目 (logEntries[]) | 组件ref | []；打开日志弹窗时 GET /logs；执行中任务通过 SSE/WebSocket 实时追加 |
| 定时配置弹窗可见 (scheduleModalVisible) | 组件ref | 默认 false；点击"定时设置"按钮 |
| 定时配置数据 (scheduleConfig) | 组件ref | null；打开定时弹窗时 GET /schedule |
| 删除确认弹窗可见 (deleteModalVisible) | 组件ref | 默认 false；点击"删除"按钮 |
| 待删除任务 (deletingTask) | 组件ref | null；点击"删除"按钮时设置 |
| 加载状态 (loading) | 组件ref | 默认 false；API请求中 |
| 目标账号下拉选项 (accountOptions[]) | Pinia: `benchmarkAccountStore` | 打开新建弹窗时 GET /benchmark-accounts?brief=true |
| 执行中任务实时进度 (runningTasks) | Pinia: `collectTaskStore` | 页面挂载后轮询5s或 SSE 推送进度更新 |

### 加载策略

1. **页面挂载**：并行发起 2 个请求（tasks列表, stats统计），独立 loading
2. **骨架屏**：6行表格骨架(48px) + 4个统计数字骨架(20px)，pulse 1.5s
3. **搜索防抖**：输入后 500ms 延迟发起请求
4. **操作后刷新**：新建/执行/暂停/删除/重试后，同时刷新列表和统计摘要
5. **执行中任务进度**：
   - 前端轮询方案：5秒轮询 GET /collect-tasks，仅更新 running 状态任务的 progress
   - SSE 方案（推荐）：连接 SSE /collect-tasks/:id/progress，实时推送进度更新
6. **日志实时更新**：执行中任务的日志弹窗，通过 SSE/WebSocket 实时追加新条目
