## 2. 操作与API

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 加载人设配置 | GET | /api/v1/persona | 无 | { data: { id, display_name, age, career_background, years_of_experience, core_experience, one_line_positioning, core_philosophy, unique_selling_point, language_styles[], style_description, catchphrases: [{ type, content }], narrative_viewpoint, sample_texts[], sample_audios[], created_at, updated_at } } | 无 | 全部表单区块 |
| 保存配置 | PUT | /api/v1/persona | { display_name, age, career_background, years_of_experience, core_experience, one_line_positioning, core_philosophy, unique_selling_point, language_styles[], style_description, catchphrases: [{ type, content }], narrative_viewpoint, sample_texts[], sample_audios[] } | { data: { id, updated_at } } | persona_config(更新，全局唯一) | 保存按钮状态 + 清除全局人设缓存 |
| 上传样稿文件 | POST | /api/v1/persona/samples | FormData: { file: File, type: "text"\|"audio" } | { data: { id, url, filename, size, type } } | 样稿文件存储 | 样稿文件列表 |
| 删除样稿文件 | DELETE | /api/v1/persona/samples/:id | 无 | { success: true } | persona_config.sample_texts/sample_audios | 样稿文件列表 |
| 预览效果（AI生成示例文案） | POST | /api/v1/persona/preview | { display_name, language_styles[], catchphrases: [{ type, content }], narrative_viewpoint, core_philosophy, one_line_positioning } | { data: { preview_text, highlights: [{ type: "violation"\|"persona", start: number, end: number }] } } | 无 | 预览弹窗内容 |
| 换一段（重新生成预览） | POST | /api/v1/persona/preview | 同预览效果 | 同预览效果 | 无 | 预览弹窗示例文案 |

### AI预览接口异步处理策略

| 策略项 | 规则 |
|--------|------|
| 处理方式 | 同步等待（预览文案较短，预计 < 10秒） |
| 超时阈值 | 15 秒未返回则提示"预览生成超时，请重试" |
| 防抖 | 生成期间"预览效果"和"换一段"按钮禁用 |
| 缓存策略 | 预览结果不缓存，每次点击重新生成 |

### 人设配置全局缓存策略

| 策略项 | 规则 |
|--------|------|
| 缓存位置 | Pinia store: personaStore.config |
| 缓存TTL | 5 分钟 |
| 缓存失效 | 本页保存成功后主动清除；其他页面保存人设时通过事件总线通知清除 |
| 读取时机 | 选题工作台 AI 生成时、脚本编辑器去AI味检查时 |

## 3. 前端状态

| 数据 | 存储位置 | 初始化时机 | 更新触发 |
|------|---------|-----------|---------|
| 人设配置原始数据 | Pinia store: personaStore.config | 页面挂载时加载 | 保存成功后刷新 |
| 表单数据（基本信息） | Vue reactive: formBasic | 加载配置后初始化 | 用户编辑各输入框 |
| 表单数据（核心定位） | Vue reactive: formPosition | 加载配置后初始化 | 用户编辑输入框/增删卖点 |
| 风格标签选择 | Vue reactive: selectedStyles | 加载配置后初始化为 language_styles[] | 用户点击预设标签或输入自定义标签 |
| 自定义风格标签列表 | Vue reactive: customTags | 初始化为 language_styles 中非预设值的项 | 用户添加/删除自定义标签 |
| 口头禅数据 | Vue reactive: catchphrases | 加载配置后初始化，按 type 分组为4个数组 | 用户在各类型组添加/删除口头禅 |
| 叙事视角 | Vue reactive: narrativePov | 加载配置后初始化 | 用户点击单选按钮 |
| 已上传样稿列表 | Vue reactive: sampleFiles | 加载配置后初始化（合并 sample_texts + sample_audios） | 上传成功/删除样稿 |
| 表单是否修改 | Vue computed: isDirty | 初始化为 false | 任何表单值与原始数据对比 |
| 保存状态 | Vue ref: saveStatus | 初始化为 "idle" | 触发保存/保存成功/保存失败 |
| 预览弹窗可见 | Vue ref: previewVisible | 初始化为 false | 点击"预览效果"/关闭弹窗 |
| 预览文案 | Vue ref: previewText | 初始化为 null | AI预览接口返回 |
| 预览高亮标注 | Vue ref: previewHighlights | 初始化为 [] | AI预览接口返回 |
| 预览生成状态 | Vue ref: previewStatus | 初始化为 "idle" | 触发预览/预览返回/超时 |

## 4. 状态流转

```
页面加载
  └── GET 加载人设配置
        │
        ├── 有数据 ──▶ 编辑态（表单填充已有数据）
        │                 ├── 各区块表单可自由编辑
        │                 ├── isDirty 检测变更
        │                 └── 表单修改后保存按钮闪烁提示
        │
        └── 无数据 ──▶ 空白态（表单全空）
                          └── 同编辑态，表单全空等待填写
                              │
                              ▼
用户编辑表单（始终可编辑）
  ├── 称呼/年龄/职业/从业年限/核心经历 → 直接编辑
  ├── 一句话定位/核心理念 → 直接编辑
  ├── 独特卖点 → 添加/删除列表项
  ├── 语言风格标签 → 点击预设标签切换/输入自定义标签
  ├── 口头禅 → 各类型组独立添加/删除
  ├── 叙事视角 → 单选切换
  └── 样稿 → 拖拽/点击上传，或删除已有文件
        │
        ▼ 用户点击"保存"
保存流程
  ├── 前端验证：称呼(display_name)不能为空
  ├── 前端验证：一句话定位(one_line_positioning)不能为空
  ├── [验证失败] → 滚动到第一个错误字段，显示错误提示
  └── [验证通过] → PUT 保存配置
        │
        ├── 保存中 → 按钮显示 spinner + "保存中..."
        ├── 保存成功 → "已保存" ✓ + $status-success，2s 恢复；清除全局人设缓存
        └── 保存失败 → "保存失败" + $status-error
              │
              ▼ 用户点击"预览效果"
预览流程
  ├── 前端验证：称呼 + 一句话定位不能为空
  ├── [验证失败] → 提示"请先填写称呼和一句话定位"
  └── [验证通过] → 弹出预览弹窗 + POST 生成示例文案
        │
        ├── 生成中 → 显示加载动画
        ├── 生成完成 → 展示预览文案 + 高亮标注
        │                 ├── 用户可点击"换一段"重新生成
        │                 └── 用户点击"关闭"关闭弹窗
        └── 生成失败 → 显示错误提示
              │
              ▼ 用户未保存离开
离开拦截
  └── isDirty 为 true → 弹出确认"未保存的更改，确定离开？"
```

## 5. 跨页面参数

| 关联页面 | 关联方式 | 传出参数 | 接收参数 |
|---------|---------|---------|---------|
| 仪表盘 | Sidebar导航进入 | 无 | 无 |
| 选题工作台 | 被引用方（AI生成时读取） | 无 | 无（选题工作台通过 GET /api/v1/persona 读取） |
| 脚本编辑器 | 被引用方（去AI味引擎引用） | 无 | 无（脚本编辑器通过 GET /api/v1/persona 读取） |

### 全局人设配置共享机制

人设配置为全局唯一记录（单例模式），多个页面通过 API 读取。保存后需通知所有缓存方失效：

```
人设配置保存成功
  ├── 清除 personaStore.config 缓存
  ├── 触发全局事件 bus:emit('persona-config-updated')
  └── 其他已打开页面（选题工作台/脚本编辑器）监听事件，清除本地缓存
```

## 6. 待确认项

| 编号 | 待确认项 | 影响范围 | 建议 |
|------|---------|---------|------|
| 1 | language_styles 中自定义标签如何持久化？ | 数据模型中 language_styles 为枚举数组 | 建议 language_styles 存储为字符串数组，后端不限制枚举值，预设标签仅作为前端快捷选项 |
| 2 | style_description 字段需新增 | 表单中有"风格描述"多行文本框，与 language_styles（枚举数组）是两个独立字段 | 已确认需在 persona_config 表新增 style_description TEXT 字段，存储自由文本描述，与 language_styles 互不替代 |
| 3 | catchphrases 的存储格式确认 | 4个类型组的口头禅管理 | 核心数据模型定义为 JSON[{type, content}]，建议 type 枚举为"开场"/"转折"/"揭晓"/"结束"，前端按 type 分组展示 |
| 4 | sample_texts 和 sample_audios 的合并展示 | 样稿列表需同时展示文本和音频 | 建议前端加载后合并为数组 [{ id, filename, size, type: "text"/"audio", url }]，删除时按 id 调用不同删除接口 |
| 5 | 预览效果高亮标注的精确度 | AI返回的 highlights 定位方式 | 建议后端返回字符级别的 start/end 索引，前端用 span 标签包裹高亮 |
| 6 | 人设配置全局唯一性的保证 | 是否允许无记录状态 | 建议系统初始化时自动创建空配置记录，前端加载时保证始终有数据 |

## 7. 对现有系统的影响

| 影响项 | 影响描述 | 处理方式 |
|--------|---------|---------|
| 全局缓存失效机制 | 人设保存后需通知所有使用方刷新缓存 | 建议使用前端事件总线 + Pinia store 缓存 TTL 双重保障 |
| 文件上传服务 | 样稿上传需要文件存储服务 | 建议使用对象存储（如 MinIO/S3），API 返回访问 URL |
| 预览AI接口 | 预览效果需要AI生成示例文案 | 需集成AI文案生成服务，传入人设参数，返回带高亮标注的文案 |
| 数据模型扩展 | 需确认 style_description 字段 | 可能需要 ALTER TABLE 新增字段 |
| 表单验证规则 | 必填字段（display_name, one_line_positioning）验证 | 前端实时验证 + 保存前二次验证 |
| 离开拦截 | 未保存修改离开页面需弹出确认 | 使用 Vue Router 的 beforeRouteLeave 守卫 |
| 多标签页同步 | 多个浏览器标签页打开时人设配置一致性 | 依赖保存后清除缓存 + 事件通知，不额外处理跨标签同步 |
