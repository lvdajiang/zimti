## 2. 操作与API

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 加载脚本数据 | GET | /api/v1/scripts/:id | 无 | { data: { id, topic_id, task_id, full_text, video_type, oral_ratio, estimated_duration, ai_check_result, status, created_at, updated_at } } | 无 | 全页（工具栏+文案区） |
| 加载分镜段落 | GET | /api/v1/scripts/:id/segments | 无 | { data: [{ id, script_id, segment_index, time_start, time_end, purpose, visual_instruction, audio_instruction, enhancement_instruction, oral_text, required_materials[], created_at }] } | 无 | 右栏分镜区 + 底部素材面板 |
| 保存脚本 | PUT | /api/v1/scripts/:id | { full_text, video_type, oral_ratio } | { data: { id, updated_at } } | scripts | 顶部未保存提示消失 |
| 生成导演脚本 | POST | /api/v1/scripts/:id/generate-storyboard | { full_text, video_type, content_skeleton } | { task_id: string, status: "generating" } | storyboard_segments(新增/替换) | 右栏分镜区；异步轮询 |
| 轮询导演脚本生成状态 | GET | /api/v1/scripts/:id/generate-storyboard/{task_id}/status | 无 | { status: "generating"\|"completed"\|"failed", segments?: [{ id, segment_index, time_start, time_end, purpose, visual_instruction, audio_instruction, enhancement_instruction, oral_text, required_materials[] }] } | 无 | 右栏分镜区 |
| 保存分镜段落 | PUT | /api/v1/scripts/:id/segments/{segment_id} | { time_start, time_end, purpose, visual_instruction, audio_instruction, enhancement_instruction, oral_text, required_materials[] } | { data: { id } } | storyboard_segments | 对应段落卡片 |
| 添加分镜段落 | POST | /api/v1/scripts/:id/segments | { segment_index, time_start, time_end, purpose, visual_instruction, audio_instruction, enhancement_instruction, oral_text, required_materials[] } | { data: { id, segment_index } } | storyboard_segments(新增) | 右栏分镜区 + 素材面板 |
| 删除分镜段落 | DELETE | /api/v1/scripts/:id/segments/{segment_id} | 无 | { success: true } | storyboard_segments(删除) | 右栏分镜区 + 素材面板 |
| 调整段落顺序 | PATCH | /api/v1/scripts/:id/segments/reorder | { segment_orders: [{ id, segment_index }] } | { success: true } | storyboard_segments.segment_index | 右栏分镜区 |
| 去AI味检查 | POST | /api/v1/scripts/:id/ai-check | { full_text, persona_id: number } | { data: { check_result: [{ severity, original_text, suggestion, start, end, type }] }, summary: { pass_count, warning_count, error_count } } | scripts.ai_check_result | 文案区高亮 + 去AI味面板 + 生成视频按钮状态 |
| 应用修改建议 | PATCH | /api/v1/scripts/:id/apply-suggestion | { original_text, suggestion, start, end } | { data: { full_text } } | scripts.full_text | 文案区文本 + 去AI味面板问题条状态 |
| 生成口播音频 | POST | /api/v1/scripts/:id/generate-voiceover | { segments: [{ segment_id, oral_text }] } | { task_id: string, status: "generating" } | oral_audios(新增) | 口播按钮状态；异步轮询，3秒间隔，90秒上限 |
| 轮询口播音频生成状态 | GET | /api/v1/scripts/:id/generate-voiceover/{task_id}/status | 无 | { status: "generating"\|"completed"\|"failed", audios?: [{ segment_id, audio_url, duration, generated_by }] } | 无 | 口播按钮状态 + 段落卡片音频标识 |
| 朗读预览（TTS） | GET | /api/v1/scripts/:id/preview-audio | { text: string } | 音频流（同步返回） | 无 | 朗读预览按钮状态（实时播放）；10秒超时 |
| 获取素材需求清单 | GET | /api/v1/scripts/:id/materials | 无 | { data: [{ type, description, source, status: "available"\|"needed", material_id }] } | 无 | 底部素材需求面板 |
| 确认生成视频 | PATCH | /api/v1/scripts/:id/status | { status: "confirmed" } | { data: { id, status: "confirmed" } } | scripts.status | 跳转视频预览与渲染页 |
| 获取人设配置（缓存） | GET | /api/v1/persona | 无 | { data: { id, display_name, language_styles[], catchphrases, narrative_viewpoint } } | 无 | 无（通过 personaStore 缓存读取，不直接调用API） |

### AI生成类API异步处理策略

| API | 处理方式 | 轮询间隔 | 超时提示 | 超时上限 |
|-----|---------|---------|---------|---------|
| 生成导演脚本 | 异步轮询 | 3秒 | 30秒提示"生成时间较长" | 60秒标记失败 |
| 去AI味检查 | 同步等待（预计<10秒） | 无 | 15秒提示"检查超时" | 30秒标记失败 |
| 生成口播音频 | 异步轮询，3秒间隔，90秒上限 | 3秒 | 30秒提示"音频生成时间较长" | 90秒标记失败 |
| 朗读预览TTS | 同步等待（短文本），同步返回音频流 | 无 | 10秒超时 | 10秒超时标记失败 |

### 防抖与缓存策略

| 策略项 | 规则 |
|--------|------|
| 生成导演脚本 | 按钮禁用+防抖，上次请求未完成前不可再次触发 |
| 去AI味检查 | 检查期间按钮禁用，文案修改后需重新检查 |
| 口播音频生成 | 按钮禁用，等待上次完成 |
| 人设配置缓存 | 复用全局 personaStore，TTL 5 分钟 |
| 脚本自动保存 | 每30秒检测 isDirty，自动保存到后端（静默） |
| 分镜段落编辑 | 单个段落失焦时自动保存（debounce 500ms） |

## 3. 前端状态

| 数据 | 存储位置 | 初始化时机 | 更新触发 |
|------|---------|-----------|---------|
| 脚本基本信息 | Pinia store: scriptStore.script | 进入页面时从路由参数或API加载 | 保存脚本API返回 |
| 分镜段落数组 | Pinia store: scriptStore.segments | 加载分镜段落API返回 | 生成导演脚本/增删改段落/拖拽排序 |

| 视频类型 | Pinia store: scriptStore.videoType | 初始化为路由参数预设或API数据 | 用户下拉切换 |
| 口播占比 | Pinia store: scriptStore.voiceRatio | 初始化为路由参数 voice_ratio_preset | 用户修改分镜段落后重新计算 |
| 预估时长 | Vue computed: estimatedDuration | 依赖 fullText 字数计算 | 口播文本变更时自动更新 |
| 去AI味检查结果 | Pinia store: scriptStore.aiCheckResult | 初始化为 null | 去AI味检查API返回 |
| 去AI味面板可见 | Vue ref: aiCheckPanelVisible | 初始化为 false | 点击"去AI味检查"/关闭面板 |
| 导演脚本生成状态 | Pinia store: scriptStore.generateStatus | 初始化为 "idle" | 触发生成/轮询/超时 |
| 口播音频生成状态 | Pinia store: scriptStore.audioGenStatus | 初始化为 "idle" | 触发音频生成/轮询/超时 |
| 朗读预览状态 | Vue ref: ttsPreviewStatus | 初始化为 "idle" | 触发TTS/播放结束/失败 |
| 朗读预览音频URL | Vue ref: ttsPreviewUrl | 初始化为 null | TTS接口返回 |
| 素材需求列表 | Pinia store: scriptStore.materials | 加载素材需求API返回 | 分镜段落数据变更后重新加载 |
| 素材面板展开状态 | Vue ref: materialPanelExpanded | 初始化为 false | 点击切换标签 |
| 选中分镜段落ID | Pinia store: scriptStore.activeSegmentId | 初始化为 null | 点击段落卡片 |
| 编辑中段落ID | Pinia store: scriptStore.editingSegmentId | 初始化为 null | 点击段落编辑菜单 |
| 拖拽中段落ID | Vue ref: draggingSegmentId | 初始化为 null | 拖拽开始/结束 |
| 表单是否修改 | Vue computed: isDirty | 依赖脚本数据与原始数据对比 | 任何编辑操作 |
| 未保存提示可见 | Vue ref: unsavedTipVisible | 初始化为 false | isDirty 变为 true |
| 路由参数 | Vue Router: route.query | 进入页面时解析 | 从选题工作台跳转时携带 |
| 人设配置缓存 | Pinia store: personaStore.config | 首次引用时加载 | 复用全局缓存 |

## 4. 状态流转

```
页面加载
  ├── 解析路由参数（topic_id, script_text, content_skeleton, voice_ratio）
  ├── 若有 script_id → GET 加载脚本数据 + 加载分镜段落
  └── 若无 script_id（从选题工作台首次进入）→ 用路由参数初始化
        │
        ▼
编辑态（idle）
  ├── 用户可自由编辑文案
  ├── 用户可切换视频类型
  ├── isDirty 检测变更，顶部黄色提示条
  └── 每30秒自动保存（静默）
        │
        ▼ 用户点击"生成导演脚本"
生成导演脚本流程
  ├── [无文案] → 按钮禁用，tooltip"请先输入口播文案"
  └── [有文案] → POST 触发生成
        │
        ├── 生成中 → 按钮loading+旋转图标，右栏骨架屏，每3秒轮询
        ├── [超时30秒] → 提示"生成时间较长，请稍候..."
        ├── [超时60秒] → 标记失败
        ├── 成功 → 右栏显示分镜段落卡片 fadeIn
        └── 失败 → 红色提示条"生成失败，请重试"+"重试"按钮
              │
              ▼ 生成完成后
分镜编辑态
  ├── 用户可编辑各段落的视觉/听觉/增强指令和口播文本
  ├── 用户可拖拽调整段落顺序
  ├── 用户可添加/删除段落
  ├── 用户可复制段落
  ├── 单个段落失焦自动保存（debounce 500ms）
  └── 段落变更后重新计算口播占比、预估时长、素材需求
        │
        ▼ 用户点击"去AI味检查"
去AI味检查流程
  ├── [无文案] → 按钮禁用
  └── [有文案] → POST 触发检查（同步等待）
        │
        ├── 检查中 → 按钮旋转图标
        ├── 成功 → 文案区高亮标注 + 右侧浮出结果面板
        │         ├── 无严重问题 → "生成视频"按钮可用
        │         ├── 有严重问题 → "生成视频"按钮禁用
        │         └── 用户可逐条点击"替换"应用建议
        └── 失败 → 提示"检查失败，请重试"
              │
              ▼ 用户点击"生成口播"
口播音频生成流程
  ├── [无口播文本] → 按钮禁用
  └── [有口播文本] → POST 触发生成
        │
        ├── 生成中 → 按钮loading，每3秒轮询
        ├── 成功 → 按钮变为"播放/暂停"，段落卡片显示音频标识
        └── 失败 → 提示"音频生成失败"
              │
              ▼ 用户点击"朗读预览"
朗读预览流程
  ├── POST 发送当前编辑区选中文本（或全部文本）
  └── 返回音频URL → 浏览器播放
        │
        ▼ 用户点击"生成视频 →"
生成视频流程
  ├── [脚本不完整] → 按钮灰色禁用，tooltip"脚本不完整或未通过去AI味检查"
  ├── [有严重AI味问题] → 按钮禁用，必须修改禁止词
  └── [脚本完整+检查通过] → PATCH 更新状态为"confirmed"
        │
        └── 跳转视频预览与渲染页（携带 script_id, segments[], materials[]）
              │
              ▼ 用户点击"← 返回"
返回选题工作台
  ├── [有未保存修改] → 弹出确认"有未保存的修改，确定离开？"
  └── [无修改或确认离开] → router.back()
```

## 5. 跨页面参数

| 关联页面 | 关联方式 | 传出参数 | 接收参数 |
|---------|---------|---------|---------|
| 选题工作台 | 接收跳转 | 无 | topic_id + script_id（建议先POST创建脚本再跳转） |
| 选题工作台 | "← 返回"按钮返回 | 无 | 无 |
| 人设配置 | 被引用方（去AI味引擎读取） | 无 | 无（通过 personaStore 缓存读取，不直接调用API） |
| 素材库 | 素材需求面板引用 | 无 | material_ids[]（已有素材的ID列表） |
| 视频预览与渲染 | 跳转出去 | script_id | 无 |

### 路由参数传递方式

```
选题工作台 → 脚本编辑器
  建议流程：先 POST /api/v1/scripts 创建 status="draft" 的脚本记录，获取 script_id
  路由: /editor/script?topic_id=123&script_id=456
  脚本编辑器通过 script_id 加载完整数据（文案、骨架、参数等均通过API获取）

脚本编辑器 → 视频预览与渲染
  路由: /editor/video-preview?script_id=456
  视频预览页通过 script_id 加载分镜段落数据和素材需求
```

| 3 | required_materials 的数据结构？ | 素材需求面板展示 | 建议 JSON[{ type: string, description: string, source: string, material_id: number|null }] |
| 4 | 去AI味检查的绝对禁止词句列表来源？ | 去AI味检查的规则基础 | 建议后端维护一张 system_forbidden_words 表（word, category, severity），定期更新 |
| 5 | 口播音频的 TTS 引擎选择？ | oral_audios.generated_by 字段 | 支持 edge-tts 和 elevenLabs，需确认默认引擎和切换机制 |
| 6 | 脚本自动保存的冲突处理？ | 自动保存与手动保存的冲突 | 建议使用乐观锁（updated_at 版本号），冲突时提示用户选择覆盖或合并 |
| 7 | 分镜段落删除后的 segment_index 重排？ | 删除段落后的索引更新 | 建议后端自动重排，前端删除后调用 reorder 接口 |
| 8 | 朗读预览的文本范围？ | TTS 预览播放哪些文本 | 建议支持：选中文本预览 / 当前段落口播文本 / 全文口播文本，默认预览当前选中段落 |

## 7. 对现有系统的影响

| 影响项 | 影响描述 | 处理方式 |
|--------|---------|---------|
| 新增数据表 | scripts 表、storyboard_segments 表、oral_audios 表需新建 | 建库脚本包含这些表 |
| 外键索引 | scripts.topic_id、storyboard_segments.script_id、oral_audios.script_id、oral_audios.segment_id 需建立索引 | 建库时一并创建索引 |
| TTS服务集成 | 朗读预览和口播音频生成需要TTS引擎 | 需集成 edge-tts（免费）和 elevenLabs（付费），支持切换 |
| 去AI味引擎 | 去AI味检查需要独立的后端服务 | 建议封装为独立模块，输入文案+人设配置，输出检查结果 |
| 文件存储 | 口播音频文件需要对象存储 | 复用样稿上传的存储服务（MinIO/S3） |
| 路由参数长度 | content_skeleton 为JSON，可能较长 | 建议从选题工作台跳转时先 POST 创建脚本记录，跳转后用 script_id 加载，避免URL过长 |
| 自动保存机制 | 30秒自动保存 + 段落失焦保存 | 需实现 debounce 自动保存逻辑，处理并发冲突 |
| 拖拽排序库 | 分镜段落拖拽需要前端拖拽库 | 建议使用 vuedraggable 或 @vueuse/integration 的 useSortable |
| 离开拦截 | 未保存修改离开需弹出确认 | 使用 Vue Router 的 beforeRouteLeave 守卫 + 浏览器 beforeunload 事件 |
| 跨页面人设缓存 | 去AI味检查需读取人设配置 | 通过 personaStore 缓存读取，不需要直接调用API，TTL 5 分钟 |
