## 2. 操作与API

### 2.1 数据查询类

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 获取概览数据 | GET | /api/v1/dashboard/overview | time_range(必填), start_date(custom时必填), end_date(custom时必填) | `{ total_views, total_views_trend, avg_completion, avg_completion_trend, avg_interaction_rate, avg_interaction_rate_trend, total_messages, total_messages_trend }` | 无 | 区块C概览卡片组 |
| 获取视频数据列表 | GET | /api/v1/dashboard/video-records | time_range(必填), start_date, end_date, page(默认1), page_size(默认20) | `{ items: [{ id, video_title, platform, publish_date, views, completion, bounce_rate, interaction_rate, comment_count }], total, page, page_size }` | 无 | 区块D-左表格 |
| 获取趋势数据 | GET | /api/v1/dashboard/trends | publish_record_id(必填), start_date, end_date | `{ timestamps: [], play_count: [], interaction_rate: [], completion_rate: [], bounce_rate: [] }` | 无 | 区块D-右-1趋势图 |

### 2.2 数据写入类

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 录入视频数据 | POST | /api/v1/dashboard/snapshots | publish_record_id(必填), play_count, completion_rate, three_second_bounce_rate, comment_count, private_message_count | `{ id, snapshot_at }` | 数据快照(INSERT) | 区块D-左当前行 + 区块C概览卡片 |
| 写本周总结 | POST | /api/v1/experience-logs | task_id(必填), week_number(必填), biggest_surprise, biggest_mistake, next_hypothesis, tags[] | `{ id, created_at }` | 经验日志(INSERT) | 区块D-右-3踩坑日志列表 |

### 2.3 AI分析类

## 4. 状态流转

### 4.1 时间范围筛选流转

```
页面加载 → 默认 time_range="this_week"
  ↓
用户点击标签 → 更新 time_range state
  ↓
触发概览+列表API请求（带新 time_range）
  ↓
数据返回 → 更新 overview_data + video_records
  ↓
UI 重新渲染
```

### 4.2 录入数据流转

```
用户点击某行"录入"按钮
  ↓
弹出录入弹窗 → is_record_modal_open=true, 记录 selected_record_id
  ↓
用户填写 play_count / completion_rate / bounce_rate / comment_count / private_message_count
  ↓
点击"保存" → POST /api/v1/dashboard/snapshots
  ↓
成功 → 关闭弹窗 → 刷新当前行数据 + 刷新概览卡片
失败 → 弹窗内显示错误提示
```

### 4.3 AI分析流转

```
用户点击"请求数据分析"
  ↓
前端校验：当前时间范围数据 ≥ 3条？
  ├─ 否 → 按钮禁用 + tooltip提示"数据不足"
  └─ 是 → 继续
       ↓
POST /api/v1/dashboard/ai-analysis → 返回 task_id
  ↓
状态变更为 "analyzing" → 按钮变 loading + 旋转
  ↓
轮询 GET /api/v1/dashboard/ai-analysis/{task_id}（间隔2秒，最长30秒）
  ├─ status=pending → 继续轮询
  ├─ status=completed → 更新 ai_conclusion + improvement_instructions，状态变 "completed"
  └─ 超过30秒 → 状态变 "failed"，提示"分析超时，请重试"
```

### 4.4 写本周总结流转

```
用户点击"写本周总结"
  ↓
前端校验：本周是否已有记录？
  ├─ 是 → 按钮文案变"查看本周记录"，跳转展示
  └─ 否 → 弹出弹窗 → is_weekly_modal_open=true
       ↓
用户填写 biggest_surprise / biggest_mistake / next_hypothesis
  ↓
点击"保存" → POST /api/v1/experience-logs
  ↓
成功 → 关闭弹窗 → 刷新 weekly_logs 列表（新卡片出现）
失败 → 弹窗内显示错误提示
```

### 4.5 导出周报流转

```
用户点击"导出周报"
  ↓
前端校验：当前 time_range 是否有数据？
  ├─ 否 → 提示"当前周期暂无数据可导出"
  └─ 是 → 按钮变 loading
       ↓
GET /api/v1/reports/weekly → 下载二进制文件
  ↓
完成 → 按钮恢复原状
失败 → 提示"导出失败，请重试"
```

## 6. 待确认项

| # | 问题 | 影响范围 | 建议 |
|---|------|---------|------|
| 1 | 概览卡片趋势对比的"上一周期"定义：today 对比 yesterday？this_week 对比 last_week？ | 区块C趋势计算 | 建议：today vs yesterday, this_week vs last_week, this_month vs last_month |
| 2 | 数据快照自动采集频率和触发机制 | 数据新鲜度 | 建议：已发布视频每天自动采集1次，运营者可手动补录 |
| 3 | AI分析结论是否持久化存储？下次进入是否显示上次分析结果？ | 区块D-右-2 分析状态恢复 | 建议：分析结果持久化，按 time_range 缓存，变更时间范围后需重新分析 |
| 4 | 周记"用到下期"是否直接创建新任务，还是追加到现有未完成任务？ | 经验日志→选题工作台流程 | 建议：追加到当前进行中的任务的参考信息区域 |
| 5 | 导出周报的文件格式：PDF 还是 Excel？ | 区块B导出按钮 | 建议：默认 Excel，支持可选 PDF |
| 6 | 自定义时间范围超过90天是否强制限制还是仅提示？ | 区块B时间筛选 | 建议：仅提示警告，不强制限制 |

## 7. 对现有系统的影响

| 影响项 | 影响描述 | 应对措施 |
|--------|---------|---------|
| 数据快照表 | 需要支持按时间范围聚合查询（SUM/AVG） | 确保在 publish_record_id + snapshot_at 上建立复合索引 |
| 平台发布记录表 | 需要关联查询视频成品获取 cover_url | 确保在 video_product_id 上有外键索引 |
| AI分析接口 | 异步任务需要任务队列支持 | 后端需引入异步任务机制（如 Bull Queue），超时30秒 |
| 经验日志表 | 每周限一条的约束 | API 层校验：查询当前周是否已有记录，有则拒绝创建 |
| 导出功能 | 需要生成文件并触发下载 | 后端引入 PDF/Excel 生成库（如 ExcelJS） |
| 跨页面通信 | "应用到下期"需要跨页面传递数据 | 使用 Pinia store 或 URL 参数传递 improvement_instructions |
