## 2. 操作与API

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 加载发布工作台数据 | GET | /api/v1/video-products/{video_product_id}/publish-workspace | 无 | { video: Video, script: Script, publish_records: PublishRecord[], seo_score?: number, seo_checks?: SeoCheck[] } | 无 | 整个页面 |
| AI生成平台文案 | POST | /api/v1/publish-records/{record_id}/generate-copy | { platform, conversion_type, script_text, persona_style? } | { title: string, description: string, tags: string[], comment_guide: string[], reply_templates: string[] } | 无(返回建议内容) | 标题/描述/标签/评论区 |
| 批量生成文案(全部平台) | POST | /api/v1/video-products/{video_product_id}/generate-copy/batch | { conversion_type, script_text, persona_style? } | { records: { platform, title, description, tags[], comment_guide[], reply_templates[] }[] } | 无(返回建议内容) | 全部平台文案 |
| 换一批(重新生成) | POST | /api/v1/publish-records/{record_id}/generate-copy | 同"AI生成平台文案"，增加 { variation: true } | 同上 | 无 | 当前平台文案区 |
| 保存平台文案 | PUT | /api/v1/publish-records/{record_id}/content | { title?, description?, tags?[] } | { publish_record: PublishRecord } | 平台发布记录.title/description/tags | 对应平台文案区 |
| SEO检查 | POST | /api/v1/publish-records/{record_id}/seo-check | { title, description, tags[], platform, conversion_type } | { seo_score: number, checks: { name, status, score, weight, suggestion? }[] } | 平台发布记录.seo_score | SEO评分圆环+检查项列表 |
| 自动保存文案 | PUT | /api/v1/publish-records/{record_id}/auto-save | { title?, description?, tags?[] } | { saved: boolean } | 平台发布记录字段 | 无(静默保存) |
| 添加话题标签 | POST | /api/v1/publish-records/{record_id}/tags | { tag: string } | { tags: string[] } | 平台发布记录.tags | 标签区 |
| 删除话题标签 | DELETE | /api/v1/publish-records/{record_id}/tags/{tag} | 无 | { tags: string[] } | 平台发布记录.tags | 标签区 |
| 复制文案 | 无(前端操作) | — | — | — | 无 | 无(剪贴板操作) |
| 生成GEO图文 | POST | /api/v1/publish-records/{record_id}/generate-geo | { script_text, tags[] } | { geo_items: { question, answer, is_citable }[] } | 平台发布记录.geo_content | GEO图文区 |
| 标记已发布 | PUT | /api/v1/publish-records/{record_id}/publish | { aigc_confirmed: boolean } | { publish_record: PublishRecord } | 平台发布记录.publish_status→已发布, published_at→now；同时触发 POST /api/v1/content-assets/auto-create（创建内容资产） | 发布状态区 |
| 确认AIGC标识 | PUT | /api/v1/publish-records/{record_id}/aigc-confirm | 无 | { aigc_reminded: true } | 平台发布记录.aigc_reminded | AIGC提醒条状态 |
| 切换平台(加载文案) | GET | /api/v1/publish-records?video_product_id={video_product_id}&platform={platform} | 无 | { publish_record: PublishRecord } | 无 | 文案区全部内容 |
| 获取转化目标 | GET | /api/v1/tasks/{task_id}/conversion-goal | 无 | { conversion_type: string } | 无 | 转化目标标签组 |
| 保存转化目标 | PUT | /api/v1/publish-records/{record_id}/conversion-type | { conversion_type: string } | { publish_record: PublishRecord } | 平台发布记录.conversion_type | 转化目标标签 |

## 4. 状态流转

### 4.1 页面初始化状态流转

```
路由进入(video_product_id) → 加载发布工作台数据(GET /publish-workspace) →
构建publish_records_map(按platform索引) → 加载默认平台(小红书)的文案 →
加载转化目标(GET /conversion-goal) → 页面就绪
                   ↘ 无视频数据 → 显示空状态"请先完成视频渲染"
```

### 4.2 平台切换状态流转

```
切换平台标签 → 检查当前平台has_unsaved_changes →
有未保存 → 自动保存当前平台文案(PUT /auto-save) → 切换到目标平台 → 从publish_records_map加载目标平台数据
无未保存 → 直接切换 → 加载目标平台数据
```

- 切换后更新：platform_title, platform_description, tags, seo_score, seo_checks, comment_guide_items, reply_template_items, geo_items, publish_status

### 4.3 AI生成文案状态流转

```
点击"换一批" → 按钮禁用+防抖 → POST /generate-copy({variation: true}) →
加载骨架屏 → API返回(title, description, tags, comment_guide, reply_templates) →
更新文案区+评论区 → fadeOut骨架屏/fadeIn新内容
                     ↘ 生成失败 → 提示"生成失败，请重试"
```

### 4.4 SEO检查状态流转

```
点击"SEO检查" → 按钮loading → POST /seo-check →
评分计算(关键词覆盖(30%) + 标题质量(25%) + 正文结构(20%) + 互动引导(15%) + 标签使用(10%)) →
返回seo_score + seo_checks[] → 更新评分圆环(动画) + 更新检查项列表
                              ↘ 检查失败 → 提示"SEO检查失败，请重试"

评分与发布联动：
seo_score ≥ 40 → "标记已发布"按钮可用
seo_score < 40 → "标记已发布"按钮禁用，tooltip "SEO评分过低，请优化后再发布"
aigc_reminded = false → 弹出AIGC提醒，用户确认后允许继续发布（不阻止）
```

### 4.5 话题标签操作状态流转

```
添加标签 → 输入标签文本 → 检查平台限制(小红书≤10/抖音≤5/视频号≤10) →
未超限 → POST /tags → tags更新 → "+"按钮保留
超限 → "+"按钮消失，阻止添加

删除标签 → 点击"×" → DELETE /tags/{tag} → tags更新 → 若低于上限"+"按钮恢复
```

### 4.6 标记已发布状态流转

```
点击"标记已发布" → 检查seo_score →
seo_score < 40 → 按钮禁用，拦截
seo_score ≥ 40 → 检查aigc_confirmed →
  未确认 → 弹出提醒"请确认已添加AI生成内容标识" → 用户确认后继续
  已确认 → PUT /publish → 更新publish_status为"已发布"，published_at为当前时间 → 按钮变为绿色"已发布"标签
```

### 4.7 GEO图文生成状态流转

```
点击"生成图文" → 按钮loading → POST /generate-geo({script_text, tags}) →
API返回geo_items[] → 卡片fadeIn逐个展示
                     ↘ 生成失败 → 提示"图文生成失败" + 重试按钮
```

### 4.8 文案自动保存状态流转

```
编辑标题/描述/标签 → has_unsaved_changes = true → 防抖2秒 →
PUT /auto-save → has_unsaved_changes = false → 静默保存成功
                                    ↘ 保存失败 → 不提示，下次修改时重试
```

## 5. 跨页面参数

| 关联页面 | 关联方式 | 传出参数 | 接收参数 |
|---------|---------|---------|---------|
| 视频预览与渲染 | 进入时接收数据 | 无 | video_product_id, video_url, script_text（通过路由参数传入） |
| 仪表盘（任务中心） | 导航进入 | 无 | 无 |
| 选题工作台 | 读取转化目标 | 无 | 读取 conversion_goal（通过task_id查询） |
| 人设配置 | AI生成文案时读取 | 无 | 读取 persona_config（语言风格，通过系统配置API获取） |
| 数据监控面板 | 发布后跳转 | video_product_id（路由跳转） | 无（数据监控面板通过video_product_id查询发布记录和数据快照） |
| 素材库 | 视频播放器展示 | 无 | video_url（视频已入库到素材库） |
| 内容资产库 | 发布后自动创建资产 | video_product_id, platform, publish_record_id | 无（后端通过 POST /api/v1/content-assets/auto-create 自动创建资产） |

## 6. 待确认项

| # | 待确认项 | 影响范围 | 建议 |
|---|---------|---------|------|
| 1 | AI文案生成的具体模型和接口（GPT-4/Claude/自训练模型） | AI生成文案API的请求/响应格式 | 后端抽象统一接口，支持模型切换 |
| 2 | SEO评分算法各维度的具体计算规则 | SEO检查API的评分逻辑 | 按权重实现：关键词覆盖(30%)+标题质量(25%)+正文结构(20%)+互动引导(15%)+标签使用(10%) |
| 3 | 各平台的字数限制是否需要动态配置 | 标题30字/描述500字的硬编码或配置化 | 建议按平台配置，不同平台限制可能不同 |
| 4 | "标记已发布"是否需要与平台API真实对接 | 当前设计为手动标记，后续是否自动发布 | 第一阶段手动标记，后续迭代自动发布API对接 |
| 5 | 评论运营建议(comment_guide/reply_templates)的数据来源 | 是纯AI生成还是结合历史数据 | 首次由AI生成，用户可编辑后存储到平台发布记录 |
| 6 | GEO图文的存储格式 | geo_content的JSON结构 | 建议存储为 { items: [{ question, answer, is_citable }] } |
| 7 | 人设配置(persona_config)的获取方式 | AI生成文案时如何注入语言风格 | 通过系统配置API获取，作为AI生成的system prompt的一部分 |
| 8 | 数据快照的自动采集机制 | 发布后多久开始采集、采集频率 | 建议发布后1小时开始，每日采集一次，连续7天 |
| 9 | 多版本视频的发布记录是否共用 | "一鱼多吃"生成的各版本是否独立发布记录 | 建议各版本独立创建发布记录，通过video_product_id关联 |

## 7. 对现有系统的影响

| 影响项 | 影响描述 | 应对方案 |
|-------|---------|---------|
| 数据库新增表 | 平台发布记录表(publish_records)、数据快照表(data_snapshots) | 新建迁移文件，video_product_id外键需建索引 |
| AI服务依赖 | AI文案生成、SEO评分、GEO图文生成均依赖AI模型 | 后端封装统一AI服务接口，增加降级和重试策略 |
| SEO评分模块 | 需要实现SEO评分算法(5维度加权计算) | 后端独立SEO评分服务，可配置权重 |
| 自动保存机制 | 文案编辑需实时防抖自动保存 | 前端防抖2秒触发，后端upsert语义 |
| 多平台文案管理 | 同一视频3个平台各自维护标题/描述/标签/发布状态 | publish_records表按platform分区，前端用Map索引 |
| 转化目标传播 | 转化目标从选题工作台传递到发布工作台 | 通过task_id查询关联的conversion_type |
| 字数限制校验 | 不同平台标题/描述/标签数量限制不同 | 前端实时校验+后端校验双重保障 |
| 发布状态一致性 | 手动标记"已发布"后的状态同步 | 发布状态由运营者手动标记，后续迭代对接平台API |
| GEO内容存储 | geo_content作为JSON存储在publish_records表 | 需确保JSON结构稳定，后续可扩展 |
| AIGC合规提醒 | 各平台对AI生成内容的标识要求不同 | 第一阶段统一提醒，后续按平台差异化处理 |
