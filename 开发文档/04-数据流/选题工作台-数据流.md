## 2. 操作与API

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 加载对标视频列表 | GET | /api/v1/viral-videos | ?ids=1,2,3（批量获取已选视频）或 ?task_id={taskId}&limit=20（按任务筛选） | { data: [{ id, title, thumbnail_url, source_platform }] } | 无 | 左栏-对标视频区 |
| 加载热点推荐列表 | GET | /api/v1/hotspots/recommended | ?relevance_score_min=50&limit=10 | { data: [{ id, title, heat_value, relevance_score, ai_suggestion, usage_status }] } | 无 | 左栏-热点推荐区 |
| AI生成选题方案 | POST | /api/v1/topic-proposals/generate | { video_ids: number[], hotspot_ids: number[], persona_id: number } | { task_id: string, status: "generating" } | topic_proposals(新增) | 中栏选题区+标题行状态；异步轮询 |
| 轮询AI生成状态 | GET | /api/v1/topic-proposals/generate/{task_id}/status | 无 | { status: "generating"\|"completed"\|"failed", proposals?: [{ id, title, benchmark_sources, hot_topic_connections, core_hook, content_skeleton, visual_highlights, ai_suggestion, oral_ratio, status }] } | 无 | 中栏选题区+标题行状态 |
| 选用选题 | PATCH | /api/v1/topic-proposals/{topic_id}/select | { action: "select"\|"deselect" } | { data: { id, status } } | topic_proposals.status | 中栏卡片状态+右栏骨架区 |
| 融合选题 | POST | /api/v1/topic-proposals/merge | { topic_ids: number[] } | { data: { id, title, benchmark_sources, core_hook, content_skeleton, visual_highlights, ai_suggestion, oral_ratio, status: "merged" } } | topic_proposals(新增融合方案) | 中栏+右栏 |
| 换一批（重新生成） | POST | /api/v1/topic-proposals/generate | 同AI生成选题方案 | 同AI生成选题方案 | topic_proposals(替换) | 中栏+右栏 |
| 确认进入脚本 | POST | /api/v1/scripts | { topic_id: number, full_text: string, content_skeleton: json, voice_ratio: number } | { data: { id, topic_id, status: "draft" } } | scripts(新增) | 跳转脚本编辑器 |
| 获取人设配置（缓存） | GET | /api/v1/persona | 无 | { data: { id, display_name, language_styles, core_philosophy, unique_selling_point, catchphrases, narrative_viewpoint, one_line_positioning } } | 无 | 无（内部引用，不需要单独UI展示） |

### AI生成API异步处理策略

| 策略项 | 规则 |
|--------|------|
| 触发方式 | 客户端 POST 触发生成，返回 task_id |
| 轮询间隔 | 前端每 3 秒轮询一次状态接口 |
| 超时阈值 | 30 秒内未完成，提示"生成时间较长，请稍候..." |
| 超时上限 | 60 秒仍未完成，标记失败，提示"生成超时，请重试" |
| 防抖 | 按钮禁用，上次请求未完成前不可再次触发 |
| 缓存策略 | 人设配置在前端 Pinia store 缓存，TTL 5 分钟，保存时清除缓存 |

## 3. 前端状态

| 数据 | 存储位置 | 初始化时机 | 更新触发 |
|------|---------|-----------|---------|
| 对标视频列表 | Pinia store: topicStore.benchmarkVideos | 页面挂载时加载 | 加载对标视频API返回 |
| 热点推荐列表 | Pinia store: topicStore.hotTopics | 页面挂载时加载 | 加载热点推荐API返回 |
| 已选对标视频ID | Pinia store: topicStore.selectedVideoIds | 初始化为空数组 | 点击对标视频卡片（切换选中） |
| 已选热点ID | Pinia store: topicStore.selectedHotspotIds | 初始化为空数组；若有路由参数 hotspot_ids[] 则初始化为该值；若路由含 ai_suggestions[] 则合并为 ai_suggestion 供生成时使用 | 点击热点卡片（切换选中） |
| 选题方案列表 | Pinia store: topicStore.proposals | 初始化为空数组 | AI生成完成/融合完成 |
| 已选用选题ID | Pinia store: topicStore.selectedProposalId | 初始化为 null | 点击"选用"按钮 |
| AI生成状态 | Pinia store: topicStore.generateStatus | 初始化为 "idle" | 触发生成/轮询结果/超时 |
| AI生成任务ID | Pinia store: topicStore.generateTaskId | 初始化为 null | 触发生成API返回 |
| 脚本骨架数据 | Pinia store: topicStore.scriptSkeleton | 初始化为 null | 选用选题后从 proposals 中提取 |
| 人设配置缓存 | Pinia store: personaStore.config | 首次引用时加载，TTL 5min | 人设配置保存事件清除 |
| 融合模式标记 | Pinia store: topicStore.isMerging | 初始化为 false | 点击"融合选题" |
| 路由参数 | Vue Router: route.query | 进入页面时解析 | 从爆款视频库/视频详情/热点追踪跳转时携带 video_ids[], hotspot_ids[], hotspot_titles[], ai_suggestions[] |

## 4. 状态流转

```
页面加载
  ├── 并行请求：对标视频列表 + 热点推荐列表 + 人设配置缓存
  └── 解析路由参数（video_ids[] / hotspot_ids[] / hotspot_titles[] / ai_suggestions[] → 合并为 ai_suggestion）
        │
        ▼
素材选择态（idle）
  ├── 用户选择对标视频（1-5个）
  ├── 用户选择热点（0-N个）
  └── 素材变更后，若已有生成结果 → 提示"素材已变更，建议重新生成"
        │
        ▼ 用户点击"换一批" 或 素材选择后自动触发
生成中态（generating）
  ├── 标题行显示"AI 正在分析素材..."
  ├── 中栏显示3个骨架屏
  ├── 前端每3秒轮询状态
  ├── [超时30秒] → 提示"生成时间较长，请稍候..."
  └── [超时60秒] → 标记失败
        │
        ├── 成功 ──▶ 生成完成态（completed）
        │               ├── 标题行显示"生成完成"
        │               ├── 中栏显示3张选题卡片
        │               └── 用户可选用/融合
        │
        └── 失败 ──▶ 生成失败态（failed）
                        ├── 标题行显示"生成失败，点击重试"
                        └── 中栏显示错误提示+"重试"按钮
                              │
                              ▼
选题选用态
  ├── 用户点击"选用" → 卡片变为已选用，右栏展示脚本骨架
  ├── 用户可切换选用（单选切换）
  └── 选用≥2个 → "融合选题"按钮可用
        │
        ▼ 用户点击"融合选题"
融合态
  ├── 被融合卡片边框变紫色虚线
  ├── 合并为新卡片（紫色虚线边框）
  ├── 右栏自动刷新为融合后骨架
  └── 用户可重新选用/换一批
        │
        ▼ 用户点击"编辑脚本 →"
确认进入脚本
  ├── POST 创建脚本记录（status: draft）
  └── 跳转脚本编辑器（携带 topic_id, script_text, content_skeleton, voice_ratio）
```

## 5. 跨页面参数

| 关联页面 | 关联方式 | 传出参数 | 接收参数 |
|---------|---------|---------|---------|
| 爆款视频库 | 接收跳转 | 无 | video_ids[]（对标视频ID数组，1-5个）；通过路由参数 `$route.query.video_ids` 接收 |
| 视频详情 | 接收跳转 | 无 | video_ids[] + video_titles[]（数组格式，即使单个视频也用数组）；通过路由参数接收 |
| 热点追踪 | 接收跳转 | 无 | hotspot_ids[]（热点ID数组）+ hotspot_titles[]（标题数组）+ ai_suggestion（合并后的单值字符串，由前端将 ai_suggestions[] 合并） |
| 仪表盘 | Sidebar导航 | 无 | 无 |
| 脚本编辑器 | 跳转出去 | topic_id, topic_title, script_text, content_skeleton, voice_ratio | 无 |
| 人设配置 | 读取引用 | 无 | 无（通过 API GET /api/v1/persona 读取全局人设配置，供AI生成时使用） |
| 数据监控面板 | 接收跳转 | 无 | improvement_instructions[], experience_log |

## 6. 待确认项

| 编号 | 待确认项 | 影响范围 | 建议 |
|------|---------|---------|------|
| 1 | 选题方案生成时是否需要关联 task_id？ | topic_proposals 表结构 | 核心数据模型中有 task_id 外键，需确认 task 的来源（仪表盘创建？还是选题工作台自动创建？） |
| 2 | 热点匹配度 relevance_score 的计算方式？ | 热点推荐列表的 match_pct 显示 | 建议由后端在 hot_topics 表中预计算，前端直接读取 |
| 3 | 融合选题后原选题方案是否保留？ | 融合操作的业务逻辑 | 建议保留原方案（status 标记为"已融合"），融合结果作为新方案插入 |
| 4 | "换一批"是否清除之前的生成结果？ | 换一批操作的数据处理 | 建议软删除旧方案（status → "已废弃"），而非物理删除 |
| 5 | benchmark_videos 表是否独立于 topic_proposals？ | 数据模型设计 | 页面描述中 benchmark_sources 是 JSON 字段存在 topic_proposals 中，但左侧列表需要独立加载；实际使用 viral_videos 表（爆款视频表）作为对标视频来源，通过 GET /api/v1/viral-videos?ids=1,2,3 批量获取 |
| 6 | 选用选题是单选还是可多选后取最后一个？ | 选用操作的交互逻辑 | 页面描述显示单选（右栏只显示一个骨架），需确认 |

## 7. 对现有系统的影响

| 影响项 | 影响描述 | 处理方式 |
|--------|---------|---------|
| 新增数据表 | topic_proposals 表需新建，含 benchmark_sources JSON 字段和 hot_topic_connections JSON 字段 | 建库脚本需包含此表 |
| 外键索引 | topic_proposals.task_id 需建立索引 | 建库时一并创建索引 |
| 人设配置缓存 | 本页面需读取人设配置供AI生成使用 | 复用全局 personaStore，通过 GET /api/v1/persona 获取，TTL 5 分钟 |
| 脚本表写入 | 确认进入脚本时需创建 scripts 记录 | scripts 表需提前创建 |
| 异步任务机制 | AI生成选题为异步操作，需后台任务队列 | 建议使用 Bull/BullMQ 任务队列，配合 Redis 存储任务状态 |
| 路由参数 | 需支持 query 参数 video_ids[], hotspot_ids[], hotspot_titles[], ai_suggestions[] | Vue Router 配置需允许这些数组参数 |
| 爆款视频库/视频详情 | 需提供"加入分析"/"用作对标"按钮，传递 video_ids[] + video_titles[] | 需协调爆款视频库和视频详情页面开发 |
| 热点追踪 | 需提供"用于选题"按钮，传递 hotspot_ids[] + hotspot_titles[] + ai_suggestions[] | 需协调热点追踪页面开发 |
