## 2. 操作与API

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 获取账号列表 | GET | /api/v1/benchmark-accounts | platform (可选), keyword (可选), page, page_size | { items: [{id, account_name, platform, platform_uid, avatar_url, follower_count, content_direction, monitor_status, last_collected_at}], total } | 无 | 区块B统计、区块D卡片列表 |
| 添加账号 | POST | /api/v1/benchmark-accounts | { account_name, platform, homepage_url, follower_count, content_direction } | { id, account_name, platform, ... } | INSERT benchmark_accounts | 区块B统计、区块D卡片列表 |
| 编辑账号 | PUT | /api/v1/benchmark-accounts/{id} | { account_name, platform, homepage_url, follower_count, content_direction } | { id, account_name, platform, ... } | UPDATE benchmark_accounts | 区块D对应卡片 |
| 删除账号 | DELETE | /api/v1/benchmark-accounts/{id} | — | { success: true } | DELETE benchmark_accounts | 区块B统计、区块D卡片列表 |
| 切换监控状态 | PATCH | /api/v1/benchmark-accounts/{id}/monitor | { monitor_status: "active"/"paused" } | { id, monitor_status } | UPDATE benchmark_accounts.monitor_status | 区块D对应卡片（状态指示点+文字+开关） |
| 立即采集 | POST | /api/v1/collect-tasks | { target_account_id, task_type: "video_list", max_count: 100 } | { id, status, ... } | INSERT collect_tasks + UPDATE benchmark_accounts.monitor_status→'collecting' | 区块D对应卡片（状态变为采集中） |
| 重试采集 | POST | /api/v1/collect-tasks/{id}/retry | — | { id, status } | UPDATE collect_tasks.status + UPDATE benchmark_accounts.monitor_status→'collecting' | 区块D对应卡片（状态变为采集中） |
| 获取账号统计摘要 | GET | /api/v1/benchmark-accounts/stats | — | { total, monitoring, paused } | 无 | 区块B统计项 |

## 4. 状态流转

### 监控状态流转

```
开启(active) → 暂停(paused) → 开启(active)
开启(active) → 采集中(collecting) → 开启(active) / 采集失败(error)
采集失败(error) → 采集中(collecting) [重试]
采集中(collecting) → 采集失败(error) [采集异常]
```

允许的转换:
- active ↔ paused：运营者手动切换监控开关（双向）
- active → collecting：系统自动定时采集 或 运营者点击"立即采集"
- collecting → active：采集成功完成
- collecting → error：采集异常（平台反爬、网络错误等）
- error → collecting：运营者点击"重试采集"

触发条件:
- 监控开关切换：运营者手动操作，采集中(collecting)状态不可切换
- 立即采集：仅 active 状态可用，paused/collecting 状态按钮禁用
- 重试采集：仅 error 状态显示重试按钮

### 采集任务状态流转（关联）

```
等待中(waiting) → 执行中(running) → 已完成(completed) / 失败(failed)
失败(failed) → 执行中(running) [重试]
```

## 7. 对现有系统的影响

- **新增API**:
  - GET /api/v1/benchmark-accounts — 获取对标账号列表（支持平台筛选+搜索+分页）
  - POST /api/v1/benchmark-accounts — 添加对标账号
  - PUT /api/v1/benchmark-accounts/{id} — 编辑对标账号
  - DELETE /api/v1/benchmark-accounts/{id} — 删除对标账号（保留关联采集数据）
  - PATCH /api/v1/benchmark-accounts/{id}/monitor — 切换监控状态
  - GET /api/v1/benchmark-accounts/stats — 获取账号统计摘要
  - POST /api/v1/collect-tasks — 创建采集任务（立即采集触发）
  - POST /api/v1/collect-tasks/{id}/retry — 重试失败采集
- **新增/修改表**:
  - benchmark_accounts 表：需包含 account_name, platform, platform_uid, avatar_url, follower_count, content_direction, monitor_status, last_collected_at, homepage_url 字段
  - 索引：platform (筛选), monitor_status (统计), user_id (归属)
  - 外键：collect_tasks.target_account_id → benchmark_accounts.id（需索引）
- **影响的页面**:
  - 数据采集任务：接收 account_id 用于创建采集任务
  - 爆款视频库：接收 account_id, platform 用于筛选该账号视频
  - 仪表盘：对标新增爆款统计依赖采集数据
