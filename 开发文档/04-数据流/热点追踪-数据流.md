## 1. 显示字段

### 1.1 热点卡片（区块E）

| 页面元素 | 字段名 | 来源类型 | 具体来源 | 备注 |
|---------|--------|---------|---------|------|
| 热点标题 | title | 📥存储 | 热点信息.title | 最多2行省略 |
| 来源标签 | source_platform | 📥存储 | 热点信息.source_platform | 映射：douyin→抖音热搜, xiaohongshu→小红书热搜, weibo→微博, travel_media→旅游行业媒体, 5118→5118需求图谱, manual→手动 |
| 发布时间 | published_at | 📥存储 | 热点信息.published_at | 格式 MM-DD HH:mm |
| 热度值 | heat_value | 📥存储 | 热点信息.heat_value | ≥10000显示"X.X万"；>100万显示"100万+" |
| 热度趋势 | heat_trend | 📥存储 | 热点信息.heat_trend | 映射：↑→上升(绿色), →→持平(灰色), ↓→下降(红色) |
| 相关度评分 | relevance_score | 📥存储 | 热点信息.relevance_score | 百分比+进度条；≥70%绿色, 30-69%黄色, <30%灰色+卡片opacity 0.6 |
| 有效期标签 | validity | 📊计算 | 根据热点信息.valid_until - 当前时间计算 | ≤24h→"紧急"(红色+左侧3px竖条), ≤48h→"限时"(黄色), >48h→"常青"(绿色) |
| 使用状态 | usage_status | 📥存储 | 热点信息.usage_status | 映射：unused→未使用(灰色), used→已选用(绿色), expired→已过期(灰色disabled) |
| AI结合建议 | ai_suggestion | 📥存储 | 热点信息.ai_suggestion | 展开/折叠，点击"查看建议"展开 |
| 关键词标签 | keywords | 📥存储 | 热点信息.keywords | 数组类型，装饰用 |
| 热点总数 | hotspot_total | 📊计算 | 列表API响应中的 total 字段 | 标题旁显示"共 N 条热点" |

## 2. 操作与API

### 2.1 查询类操作

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 获取热点列表 | GET | /api/v1/hotspots | page(int, 默认1), page_size(int, 默认20), source_platform(string, 可选: douyin/xiaohongshu/weibo/travel_media/5118/manual), time_range(string, 可选: today/24h/7d), relevance(string, 可选: high/medium/low), keyword(string, 可选, 按标题模糊搜索), sort_by(string, 默认heat_value: heat_value/relevance_score/published_at), order(string, 默认desc: asc/desc) | { total: number, page: number, page_size: number, items: [{ id, title, source_platform, heat_value, heat_trend, relevance_score, ai_suggestion, keywords, published_at, valid_until, usage_status }] } | 无 | 热点卡片列表 |
| 获取监控关键词列表 | GET | /api/v1/keywords | 无 | { items: [{ id, keyword, match_count, last_matched_at, created_at }] } | 无 | 关键词监控区 |
| 获取关键词趋势 | GET | /api/v1/keywords/trends | keyword_id(int, 可选), days(int, 默认7) | { items: [{ keyword, trend_data: [{ date, match_count }] }] } | 无 | 趋势折线图 |

### 2.2 操作类操作

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 刷新热点 | POST | /api/v1/hotspots/refresh | { source_platform: string (可选，指定刷新来源) } | { success: boolean, new_count: number } | 热点信息表（拉取最新数据） | 热点列表（重新加载） |
| 手动录入热点 | POST | /api/v1/hotspots | { title: string(必填), source_platform: string(必填, 枚举值含manual), source_url: string(可选), note: string(可选) } | { success: boolean, id: number } | 热点信息表新增记录 | 热点列表（插入新卡片到顶部） |
| 带入选题 | 无（前端跳转） | router.push 到选题工作台 | hotspot_ids[], hotspot_titles[], ai_suggestions[] | 无 | 无（跳转选题工作台，由工作台处理素材关联） | 当前热点卡片的使用状态标签 |
| 标记过期 | PUT | /api/v1/hotspots/:id/expire | id(路径参数, number) | { success: boolean } | 热点信息.usage_status 更新为 expired | 当前热点卡片（opacity 0.5） |
| 添加监控关键词 | POST | /api/v1/keywords | { keyword: string(必填, 唯一) } | { success: boolean, id: number } | 关键词监控表新增记录 | 关键词监控区（标签组+趋势图） |
| 删除监控关键词 | DELETE | /api/v1/keywords/:id | id(路径参数, number) | { success: boolean } | 关键词监控表删除记录 | 关键词监控区（标签组+趋势图） |

## 4. 状态流转

### 4.1 使用状态 (usage_status)

```
未使用(unused) ──(点击"用于选题")──→ 已选用(used)
       │
       │──(超过有效期未使用)──→ 已过期(expired)
       │
       └──(手动点击"已过期")──→ 已过期(expired)
```

注意：已选用/已过期不可回退到未使用。

### 4.2 有效期状态 (validity)

```
计算规则：valid_until - 当前时间
  ≤24h → 紧急（左侧红色竖条 + 红色标签）
  ≤48h → 限时（黄色标签）
  >48h → 常青（绿色标签）
  已过期 → opacity 0.5
```

### 4.3 页面加载状态

```
初始加载中（骨架屏5张）──(成功)──→ 已加载
       │
       └──(失败)──→ 错误状态 ──(重试)──→ 初始加载中
```

### 4.4 关键词匹配提醒流程

```
系统定时检测 ──(发现匹配)──→ WebSocket推送
                                  │
                                  └──→ 弹窗提醒("监控关键词「xxx」匹配到N条新热点")
                                            │
                                            ├──(查看详情)──→ 刷新列表 + 高亮匹配项
                                            │
                                            └──(稍后)──→ 关闭弹窗
```

### 4.5 手动录入流程

```
点击"手动录入" ──→ 弹窗打开 ──(填写表单)──→ 提交 ──(成功)──→ 关闭弹窗 + 列表顶部插入新卡片
                                         │
                                         └──(重复)──→ 提示"该热点已存在"
                                         │
                                         └──(失败)──→ 保留弹窗 + 错误提示
```

## 5. 跨页面参数

| 关联页面 | 关联方式 | 传出参数 | 接收参数 |
|---------|---------|---------|---------|
| 选题工作台 | "用于选题"按钮跳转（支持多选） (router.push) | hotspot_ids[]: number[], hotspot_titles[]: string[], ai_suggestions[]: string[] | 选题工作台通过路由参数接收 hotspot_ids[] + hotspot_titles[]，并将 ai_suggestions[] 合并为 ai_suggestion 单值字符串 |
| 仪表盘 | Sidebar导航 / 仪表盘快捷入口 | 无 | 无 |
| 仪表盘热点推荐 | "查看更多"链接跳转 | 无 | 无 |
| 数据采集任务 | Sidebar导航 | 无 | 无 |

### 路由定义

- 页面路径：`/hotspots`
- 选题跳转（单选）：`/workspace?hotspot_ids[]={id}&hotspot_titles[]={title}&ai_suggestions[]={suggestion}`
- 选题跳转（多选）：`/workspace?hotspot_ids[]={id1}&hotspot_ids[]={id2}&hotspot_titles[]={title1}&hotspot_titles[]={title2}&ai_suggestions[]={suggestion1}&ai_suggestions[]={suggestion2}`

## 6. 待确认项

| # | 待确认内容 | 影响范围 | 建议方案 |
|---|-----------|---------|---------|
| 1 | 关键词匹配提醒的实现方式 | 新匹配提醒弹窗 | 建议使用 WebSocket 实时推送；初期可用轮询（30秒间隔）替代 |
| 2 | 热点数据自动采集的调度策略 | 刷新热点功能 | 后端定时任务采集各平台热搜，前端"刷新"按钮触发即时采集 |
| 3 | 相关度评分(relevance_score)的计算方式 | 数据展示 | 需定义算法：可基于关键词匹配+内容方向相似度+用户行为加权 |
| 4 | AI结合建议(ai_suggestion)的生成时机 | 数据展示 | 建议在热点入库时由AI自动生成，而非前端点击时实时生成 |
| 5 | 热点过期自动标记的触发方式 | 状态流转 | 建议后端定时任务扫描，将超时未使用的热点标记为expired |
| 6 | 手动录入热点是否需要审核 | 手动录入功能 | 初期建议不需要审核，直接入库 |
| 7 | 趋势曲线图的数据来源和时间粒度 | 关键词趋势 | 建议近7天数据，按天聚合匹配数量 |
| 8 | 敏感话题过滤的实现方式 | 数据展示 | 建议后端采集时通过敏感词库过滤，前端不展示 |
