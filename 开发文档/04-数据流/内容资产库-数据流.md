## 1. 显示字段

### 区块B：页面标题行

| 页面元素 | 字段名 | 来源类型 | 具体来源 | 备注 |
|---------|--------|---------|---------|------|

| 缩略图 | thumbnail | 📥存储 | 视频成品.cover_url（通过内容资产.video_product_id 关联） | 脚本类型显示脚本图标，图文类型显示图文缩略图 |

## 2. 操作与API

### 2.1 数据查询类

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 获取资产列表 | GET | /api/v1/content-assets | type(all/脚本/视频/图文), platform(all/小红书/抖音/视频号), performance(all/high_completion/high_interaction/high_conversion), time_filter(all/recent_7d/recent_30d/archived), keyword, page(默认1), page_size(默认20) | `{ items: [{ id, title, type, thumbnail, platforms, performance_tags, custom_tags, views, interaction_rate, highlights, status, published_at }], total, page, page_size, stats: { total, script_count, video_count, image_text_count } }` | 无 | 区块B统计 + 区块D卡片列表 |
| 获取资产详情 | GET | /api/v1/content-assets/{asset_id} | 无 | `{ id, title, type, video_product_id, platforms, status, published_at, archived_at, core_metrics: { play_count, interaction_rate, completion_rate, comment_count, private_message_count }, element_highlights, custom_tags, reusable_materials, preview_url, created_at }` | 无 | 区块E详情弹窗 |
| 获取资产趋势数据 | GET | /api/v1/content-assets/{asset_id}/trends | start_date, end_date | `{ timestamps: [], play_count: [], interaction_rate: [], completion_rate: [] }` | 无 | 区块E迷你趋势图 |
| 获取可复用素材 | GET | /api/v1/content-assets/{asset_id}/materials | 无 | `{ materials: [{ material_id, name, thumbnail, copyright_status, expires_at }] }` | 无 | 区块F素材列表 |

### 2.2 数据写入类

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 标记元素亮点 | POST | /api/v1/content-assets/{asset_id}/highlights | highlight_desc(必填), highlight_tags[], custom_tags[] | `{ highlight_id, created_at }` | 内容资产.element_highlights[](追加) + 内容资产.custom_tags[](追加) | 区块D当前卡片亮点行 + 区块E标签区 |
| 复用素材到任务 | POST | /api/v1/content-assets/{asset_id}/reuse-materials | material_ids[](必填), task_id(必填) | `{ task_id, imported_count }` | 任务.关联素材(UPDATE) | 无（提示成功） |
| 复用脚本结构 | POST | /api/v1/content-assets/{asset_id}/reuse-script | task_id(必填) | `{ task_id, script_id, pasted_at }` | 脚本.full_text(追加段落) | 无（跳转脚本编辑器） |
| 删除资产 | DELETE | /api/v1/content-assets/{asset_id} | 无（仅 status=draft 允许） | `{ deleted: true }` | 内容资产(DELETE) | 区块D列表移除该卡片 |

### 2.3 导出类

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 导出资产 | GET | /api/v1/content-assets/{asset_id}/export | format(可选，默认 excel) | 二进制文件(PDF/Excel) | 无 | 无（触发下载） |

### 2.4 系统自动操作

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 发布后自动创建资产 | POST | /api/v1/content-assets/auto-create | video_product_id(必填), platform, publish_record_id | `{ asset_id }` | 内容资产(INSERT)，由发布工作台触发 | 无（后台自动） |
| 自动归档(发布30天后) | PUT | /api/v1/content-assets/{asset_id}/auto-archive | 无（定时任务触发，条件：status=published AND published_at <= NOW() - 30天） | `{ archived: true }` | 内容资产.status → 'archived', archived_at = now() | 定时任务，每天执行一次 |

## 4. 状态流转

### 4.1 资产列表筛选流转

```
页面加载 → 默认所有筛选=all, keyword="", page=1
  ↓
用户切换任意筛选标签 / 输入搜索关键词
  ↓
重置 page=1 → 合并所有筛选参数
  ↓
GET /api/v1/content-assets（带完整筛选参数）
  ↓
数据返回 → 更新 asset_list + asset_stats
  ↓
UI 重新渲染卡片列表 + 统计信息
```

### 4.2 查看详情流转

```
用户点击卡片"查看详情"按钮
  ↓
设置 selected_asset_id → is_detail_modal_open=true
  ↓
GET /api/v1/content-assets/{asset_id}
  ↓
数据返回 → 更新 asset_detail → 渲染详情弹窗
  ↓
弹窗内展示：预览区 + 标签区 + 数据区 + 素材区 + 操作按钮
```

### 4.3 复用素材流转

```
用户点击"复用素材"按钮
  ↓
GET /api/v1/content-assets/{asset_id}/materials → 获取素材列表
  ↓
弹出素材复用弹窗 → is_material_modal_open=true
  ↓
前端校验：是否有版权过期素材？
  ├─ 是 → 顶部显示红色提醒
  └─ 否 → 正常显示
       ↓
用户勾选素材 → 更新 selected_material_ids
  ↓
点击"添加到当前任务" → POST /api/v1/content-assets/{asset_id}/reuse-materials
  ↓
成功 → 关闭弹窗 + 提示"已添加 N 个素材到当前任务"
失败 → 弹窗内显示错误提示
```

### 4.4 复用脚本结构流转

```
用户点击"复用结构"按钮
  ↓
POST /api/v1/content-assets/{asset_id}/reuse-script
  ↓
成功 → 按钮文案变"已复制" → 延迟后跳转脚本编辑器
失败 → 提示"复用失败，请重试"
```

### 4.5 标记元素亮点流转

```
用户点击"标记亮点"按钮
  ↓
弹出标记亮点弹窗 → is_highlight_modal_open=true
  ↓
用户填写描述 + 选择预设标签 / 输入自定义标签
  ↓
点击"保存" → POST /api/v1/content-assets/{asset_id}/highlights
  ↓
成功 → 关闭弹窗 → 刷新卡片亮点行 + 详情弹窗标签区
失败 → 弹窗内显示错误提示
```

### 4.6 删除资产流转

```
用户点击"删除"按钮
  ↓
前端校验：asset_status === 'draft'？
  ├─ 否 → 按钮禁用，提示"已发布资产不可删除"
  └─ 是 → 弹出确认弹窗 → is_delete_modal_open=true
       ↓
用户点击"删除"确认 → DELETE /api/v1/content-assets/{asset_id}
  ↓
成功 → 卡片 fadeOut 动画移除 → 刷新 asset_stats
失败 → 提示错误信息
```

### 4.7 自动归档流转

```
定时任务（每天凌晨执行）
  ↓
查询所有 status='published' AND published_at <= NOW() - 30天 的资产
  ↓
逐条 PUT /api/v1/content-assets/{asset_id}/auto-archive
  ↓
内容资产.status → 'archived'
内容资产.archived_at → NOW()
  ↓
用户下次访问时，归档资产仅在 time_filter='archived' 时显示
```

## 6. 待确认项

| # | 问题 | 影响范围 | 建议 |
|---|------|---------|------|
| 1 | 表现筛选"高完播""高互动""高转化"的具体阈值是否固定？ | 区块C表现筛选 | 当前阈值来自需求文档（高完播>=40%、高互动>=5%、高转化>=10），建议后续配置化 |
| 2 | 搜索范围是否包含脚本全文内容？大文本搜索性能问题 | 区块B搜索框 | 建议：标题 + 标签优先搜索，脚本全文搜索使用 PostgreSQL 全文检索(GIN索引) |
| 3 | 资产创建时机：发布成功后立即创建，还是首次采集到数据后创建？ | 自动创建资产流程 | 建议：发布成功后立即创建（status=已发布），core_metrics 初始为空 |
| 4 | 复用素材时"添加到当前任务"：当前任务如何确定？ | 区块F素材复用弹窗 | 建议：获取当前进行中的任务(status≠已完成)，多个时弹出选择器 |
| 5 | 归档资产是否可以恢复为已发布状态？ | 归档逻辑 | 建议：不可恢复，归档为最终状态，但仍然可以查看和复用 |
| 6 | 素材版权过期时间的计算依据？ | 区块F版权状态 | 建议：需要在 reusable_materials 中增加 expires_at 字段，或从外部版权系统获取 |
| 7 | 内容资产与平台发布记录的关系：一个视频可能发布到多个平台，是否每个平台一条资产？ | 数据模型关联 | 建议：一个视频一条资产，publish_platforms[] 记录所有平台，core_metrics 取各平台聚合值 |
| 8 | 图文类型资产的缩略图来源？ | 区块D缩略图 | 建议：图文类型取首张图片作为缩略图，需在内容资产表中增加 cover_url 字段 |

## 7. 对现有系统的影响

| 影响项 | 影响描述 | 应对措施 |
|--------|---------|---------|
| 内容资产表 | 需要支持多维度筛选查询（类型+平台+表现+时间） | 建立 (status, type, created_at) 复合索引；平台筛选使用 PostgreSQL 数组包含操作 `@>` |
| 内容资产.core_metrics | JSON 字段的查询和聚合 | 使用 PostgreSQL JSONB 类型，对 core_metrics->play_count 和 core_metrics->interaction_rate 创建 GIN 索引 |
| 内容资产.publish_platforms | 数组字段的筛选查询 | 使用 PostgreSQL 数组类型，建立 GIN 索引支持 `@>` 操作 |
| 全文搜索 | 搜索需覆盖标题、标签、脚本全文 | 使用 PostgreSQL tsvector + GIN 索引实现全文检索 |
| 自动归档定时任务 | 发布30天后自动归档 | 后端引入定时任务（如 node-cron 或 Bull Queue repeatable job），每天检查一次 |
| 发布工作台联动 | 发布成功后自动创建资产记录 | 发布工作台的发布成功回调中调用 POST /api/v1/content-assets/auto-create |
| AI表现标签计算 | 需要聚合数据快照数据生成表现标签 | 可在每次查询资产列表时实时计算，或在数据快照更新时异步更新资产表现标签 |
| 数据监控面板跳转 | 点击资产数据跳转到数据监控面板 | 通过 video_product_id 关联，URL 参数传递 video_product_id |
| 脚本编辑器联动 | 复用结构需跳转并粘贴脚本内容 | 通过路由参数传递 script_structure，脚本编辑器接收后自动粘贴 |
