## 2. 操作与API

| 操作名称 | HTTP方法 | API端点 | 请求参数 | 响应格式 | 变更数据 | 刷新范围 |
|---------|---------|---------|---------|---------|---------|---------|
| 加载分镜数据 | GET | /api/v1/scripts/{script_id}/storyboard | 无 | { script: Script, segments: Segment[], materials: Material[], oral_audios: OralAudio[] } | 无 | 整个页面 |
| 更新片段顺序 | PUT | /api/v1/scripts/{script_id}/segments/reorder | { segment_ids: string[] (新顺序) } | { segments: Segment[] } | 分镜段落.segment_index | 时间轴 |
| 更新片段时长 | PUT | /api/v1/scripts/{script_id}/segments/{segment_id} | { time_start?, time_end? } | { segment: Segment } | 分镜段落.time_start/time_end | 时间轴+预览 |
| 更新片段转场 | PUT | /api/v1/scripts/{script_id}/segments/{segment_id}/transition | { transition_type: string } | { segment: Segment } | 分镜段落增加transition_type字段 | 时间轴转场标记 |
| 替换片段素材 | PUT | /api/v1/scripts/{script_id}/segments/{segment_id}/materials | { material_ids: string[] } | { segment: Segment, materials: Material[] } | 分镜段落.required_materials | 右侧面板素材区 |
| 更新渲染配置 | PUT | /api/v1/scripts/{script_id}/render-config | { resolution?, subtitle_size?, subtitle_color?, subtitle_position?, bgm_volume?, voice_volume?, sfx_volume?, transitions?[] } | { render_config: RenderConfig } | 脚本或视频成品的render_config JSON | 预览+右侧面板 |
| 生成预览(Remotion) | POST | /api/v1/video-products/:id/preview | { script_id, render_config: RenderConfig } | { preview_url: string, duration: number } | 无(临时预览) | 播放器 |
| 发起渲染(主版本) | POST | /api/v1/video-products/render | { script_id, resolution, render_config: RenderConfig } | { video_product_id: string, task_id: string } | 视频成品表新增记录(status=渲染中) | 渲染弹窗进度 |
| 轮询渲染状态 | GET | /api/v1/video-products/:id/render-status | 无 | { status: "rendering"/"completed"/"failed", progress: number, estimated_remaining?: number, video_url?: string, file_size?: number } | 视频成品表status字段 | 渲染进度条 |
| 取消渲染 | POST | /api/v1/video-products/:id/render-cancel | 无 | { success: boolean } | 视频成品表status更新为failed | 渲染弹窗关闭 |
| 批量生成多版本(一鱼多吃) | POST | /api/v1/video-products/:id/derive | { platforms: ["douyin"/"xiaohongshu"/"weixin"/"image_text"] } | { tasks: { platform, task_id }[] } | 视频成品表新增多条记录(主版/抖音版/视频号版/图文版) | 多版本弹窗进度 |
| 轮询多版本状态 | GET | /api/v1/video-products/:id/render-status | 无 | { tasks: { platform, status, progress, video_url? }[] } | 各平台视频成品status | 多版本弹窗各平台进度 |
| 保存当前调整 | POST | /api/v1/scripts/{script_id}/save-progress | { timeline_segments, render_config } | { saved_at: string } | 分镜段落+渲染配置 | 无(静默保存) |
| 获取Remotion合成配置 | GET | /api/v1/scripts/{script_id}/composition | 无 | { composition_id, duration_in_frames, fps, width, height, input_props: JSON } | 无 | Remotion播放器初始化 |

## 4. 状态流转

### 4.1 页面初始化状态流转

```
路由进入(script_id) → 加载分镜数据(GET /storyboard) → 构建timeline_segments →
获取Remotion合成配置(GET /composition) → 初始化Remotion播放器 → 预览就绪
                                                  ↘ 无分镜数据 → 显示引导空状态
```

### 4.2 预览播放状态流转

```
点击播放 → Remotion播放器播放 → playhead_position实时更新 → 播放头移动 → 时间轴同步滚动
         ↘ 无分镜数据 → 提示"请先完成分镜脚本"
播放中 → 点击暂停/播放到结尾 → 暂停 → 可重新播放
拖拽播放头 → playhead_position更新 → Remotion跳转到对应帧 → 视频画面更新
```

### 4.3 片段编辑状态流转

```
选中片段 → 右侧面板加载该片段素材和配置
拖拽排序 → 本地更新timeline_segments顺序 → 防抖后调用PUT /reorder → 时间轴重绘
拖拽时长手柄 → 本地更新segment的time_start/time_end → 防抖后调用PUT /segment → 时间轴+预览更新
修改转场 → 本地更新transition_type → 调用PUT /transition → 时间轴标记更新 → 预览自动重新渲染
替换素材 → 弹出素材选择器(跳转素材库) → 选择新素材 → 调用PUT /materials → 右侧面板更新
```

### 4.4 渲染导出状态流转

```
点击"渲染导出" → 弹出渲染弹窗 → 选择分辨率 → 点击开始 →
POST /video-products/render(video_product_id, task_id) → 轮询GET /video-products/:id/render-status(progress 0-100%) →
渲染完成(video_url) → 显示下载按钮 + "在素材库中查看"链接
                     ↘ 渲染失败 → 显示错误信息 + 重试按钮
                     ↘ 用户取消 → POST /video-products/:id/render-cancel → 关闭弹窗
```

### 4.5 多版本生成状态流转

```
前提：主版本必须已完成(status=已完成)，否则按钮禁用
点击"生成多版本" → 弹出多版本弹窗 → 勾选平台 → 点击批量生成 →
POST /video-products/:id/derive → 轮询GET /video-products/:id/render-status →
各平台独立进度条 → 各平台逐一完成 → 可下载
                  ↘ 部分平台失败 → 显示失败平台错误，成功平台可正常下载
```

### 4.6 返回脚本编辑器状态流转

```
点击"← 返回" → 检查has_unsaved_changes →
有未保存 → 弹出确认"是否保存当前调整？" → 保存(POST /save-progress) → 跳转回脚本编辑器
                                         → 不保存 → 跳转回脚本编辑器
无未保存 → 直接跳转回脚本编辑器
```

## 5. 跨页面参数

| 关联页面 | 关联方式 | 传出参数 | 接收参数 |
|---------|---------|---------|---------|
| 脚本编辑器 | 进入时接收数据 | 无 | script_id（通过路由参数传入，用于加载分镜数据） |
| 脚本编辑器 | 返回时 | script_id（路由跳转） | 无 |
| 素材库 | 替换素材时跳转 | material_ids[]（当前片段已有素材，用于筛选可选素材） | 替换后的material_id（返回时通过事件/回调） |
| 发布工作台 | 渲染完成后跳转 | video_product_id, video_url, script_text | 无（发布工作台通过video_product_id加载） |
| 素材库 | 渲染完成后 | 无 | 视频成品作为素材自动入库（后端触发） |

## 6. 待确认项

| # | 待确认项 | 影响范围 | 建议 |
|---|---------|---------|------|
| 1 | Remotion服务端渲染的部署方式（Lambda/本地Docker/云函数） | 渲染API的实现和轮询机制 | 建议使用Remotion Lambda，支持并发渲染多版本 |
| 2 | 视频成品的"多版本"存储结构 | 多版本是同一video_product_id的不同记录还是子记录 | 建议每条视频成品独立记录，通过task_id和parent_video_product_id关联 |
| 3 | 预览的实时性要求（修改后是否立即重新渲染预览） | 预览API调用频率和性能 | 建议修改后防抖2秒自动重新预览，避免频繁调用 |
| 4 | 时间轴拖拽的精度要求（帧级别/秒级别） | 拖拽交互的吸附逻辑 | 建议秒级精度，吸附到整秒，Remotion以30fps计算 |
| 5 | 渲染超时的阈值设定 | 渲染取消和降级策略 | 建议10分钟超时，超时后提示可取消 |
| 6 | 口播音频在预览中的同步方式 | Remotion中音频和视频的合成方式 | 口播音频URL作为Remotion input_props传入 |
| 7 | 片段素材缺失时是否允许渲染 | 渲染前置校验规则 | 建议素材缺失时阻止渲染并提示补充 |
| 8 | "一鱼多吃"版本裁剪规则的具体参数 | 多版本生成的差异化配置 | 4个版本：主版(原始)/抖音版(9:16裁剪)/视频号版(16:9保留)/图文版(3:4图文提取)，API端点 POST /api/v1/video-products/:id/derive |

## 7. 对现有系统的影响

| 影响项 | 影响描述 | 应对方案 |
|-------|---------|---------|
| Remotion集成 | 需要部署Remotion渲染服务，支持预览和导出 | 后端封装Remotion渲染API，前端集成Remotion Player |
| 数据库新增字段 | 分镜段落表需增加transition_type字段；视频成品表需增加parent_video_product_id(多版本关联) | 新建迁移文件，parent_video_product_id需建索引 |
| 文件存储压力 | 视频渲染产物（主版本+多版本）占用大量存储空间 | 制定存储清理策略，过期草稿视频自动清理 |
| 渲染任务队列 | 多版本渲染可能产生大量并发任务 | 使用任务队列（如Bull）管理渲染任务，控制并发数 |
| 前端Remotion依赖 | 前端需引入@remotion/player和@remotion/cli | 评估bundle size影响，考虑动态加载 |
| 素材库联动 | 渲染完成的视频需自动入库到素材库(type=视频) | 后端渲染完成后触发素材创建事件 |
| 分镜段落render_config | 需要存储每个片段/全局的渲染配置（字幕/音量/转场等） | 在视频成品表的render_config JSON字段中统一存储 |
