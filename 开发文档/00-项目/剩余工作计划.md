## 三、阶段三续：AI 服务接入（约 3-5 天）

> **前置条件**: GLM API Key + Pexels API Key

### 3A. GLM 大模型接入（1 天）
- 安装 `openai` SDK（GLM 兼容 OpenAI 格式，换 baseURL 即可）
- 创建 `GLMProvider` 实现 `AIServiceProvider` 接口
- 配置 `.env`: `GLM_API_KEY=xxx`, `GLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4`, `GLM_MODEL=glm-4-flash`
- 替换 MockAIProvider → GLMProvider
- **效果**: 选题生成、分镜生成、AI风味检查、文案生成、预览文案、视频分析全部从 Mock 变为真实可用

### 3B. Pexels 图库接入（0.5 天）
- 安装 `pexels` npm 包
- 实现 `GET /materials/pexels-search` 真实搜索
- 实现 `POST /materials/pexels-import` 下载并创建素材记录

### 3C. TTS 配音接入（1 天）
- 安装对应 TTS SDK（建议阿里云智能语音）
- 创建 `packages/server/src/services/external/tts.ts`
- 实现 `POST /scripts/:id/generate-voiceover` 异步调用 TTS
- 实现 `GET /scripts/:id/preview-audio` 短文本预览
- 配音文件上传到 MinIO/OSS

### 3D. Whisper 语音转文字（0.5 天）
- 接入 OpenAI Whisper API
- 实现 `POST /viral-videos/:id/transcript/extract` 提取视频文案

### 3E. 素材 AI 生成（0.5 天）
- 接入即梦 API（或 DALL-E / 通义万相）
- 实现 `POST /materials/generate` + 状态轮询 + 确认入库
- 生成的素材自动关联到分镜片段

### 3F. 视频分析 + 热点爬取（1 天）
- 批量视频分析：调用 GLM 分析视频元数据+文案
- 热点刷新：对接 Python MediaCrawler 微服务（:8001）

**数据模型**: 配置表或常量，支持后续扩展添加新工具

### 7.2 创作者知识库（1 天）

**核心流程**:
```
用户在 AI 工具生成内容 → 复制 → 回到 Zimti → 粘贴到知识库快收面板 → 打标签分类 → 保存
写脚本/选题时 → 搜索知识库 → 引用已有素材
```

**数据模型**:
```
KnowledgeItem {
  id:          UUID
  userId:      UUID
  title:       String(200)     — 标题（自动截取或用户自定义）
  content:     Text            — 内容全文
  source:      String(50)      — 来源（deepseek/豆包/即梦/kimi/手动...）
  category:    String(50)      — 分类（文案模板/开头钩子/金句/研究素材/脚本片段/其他）
  tags:        String[]        — 标签
  taskId:      UUID?           — 关联任务（可选）
  createdAt:   DateTime
  updatedAt:   DateTime
}
```

**API 端点**:
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/knowledge` | 列表（支持分类/标签/关键词筛选） |
| POST | `/api/v1/knowledge` | 创建 |
| PUT | `/api/v1/knowledge/:id` | 更新 |
| DELETE | `/api/v1/knowledge/:id` | 删除 |
| GET | `/api/v1/knowledge/search` | 全文搜索 |

**前端**:
- 工具导航页右侧常驻"知识库快收"面板
- 知识库管理页面（列表/分类/搜索）
- 脚本编辑器中引用知识库内容

### 7.3 放置位置
