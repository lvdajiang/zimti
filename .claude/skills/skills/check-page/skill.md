# 页面完整性检查 `/check-page`

> **使用场景**：完成页面开发后，或需要审查页面质量时调用。
> **目标**：确保页面实现与设计稿和数据逻辑文档一致。

## 执行方式（Subagent 模式）

本技能通过 Subagent 执行分析，不占用主对话上下文。

**必须使用 Agent tool** 启动分析代理，传入以下参数：
- subagent_type: "general-purpose"
- prompt: 包含下方完整检查步骤 + UI 开发规范 + 页面URL/路径

代理在独立上下文中读取设计稿（shejigao.pen）和页面代码，逐项对比，**只返回差异清单摘要给主对话**。禁止在主窗口中读取设计稿和代码。

## 用法

```
/check-page http://39.97.59.179/admin/order-pool   # 对照URL检查
/check-page admin/order-pool                        # 简写，自动拼接域名
/check-page client/resource                         # 客户端资源库
```

## 执行步骤

### 第1步：确认页面标识（5秒）

根据URL确定：
- **页面名称**：如"原始订单池"
- **对应组件**：如 `OrderPoolTable.vue`
- **设计稿位置**：在 `shejigao.pen` 中搜索对应页面名称

### 第2步：读取设计稿（30秒）

使用 Pencil MCP 工具读取 `开发文档/03-视觉/shejigao.pen` 中对应页面的设计：
- 搜索页面名称或组件名称
- 提取：布局结构、文案内容、颜色代码、组件位置、交互元素

### 第3步：读取数据逻辑文档（15秒）

读取 `开发文档/04-数据流/` 中对应的字段定义和业务规则。

### 第4步：逐项对比检查

#### 4.1 视觉一致性

| 检查项 | 内容 |
|--------|------|
| 颜色 | 主色、辅色、状态色（成功/警告/错误）是否与设计稿一致 |
| 字体 | 字号、字重是否一致 |
| 间距 | padding、margin 是否与设计稿一致 |
| 布局 | 整体布局结构（上中下、左右分栏）是否一致 |
| 图标 | 使用了正确的图标而非替代品 |

#### 4.2 功能完整性

| 检查项 | 内容 |
|--------|------|
| 按钮 | 设计稿中的所有按钮是否都已实现且功能正常 |
| 表单 | 所有表单字段是否齐全，验证规则是否正确 |
| 表格 | 列定义是否与文档一致，字段名/数据来源是否正确 |
| 分页 | 分页组件是否正常工作（不被裁切） |
| 搜索/筛选 | 筛选条件是否完整 |
| 对话框 | 编辑/新增对话框的字段和交互是否与设计一致 |

#### 4.3 数据准确性

| 检查项 | 内容 |
|--------|------|
| 字段来源 | 显示数据是否从正确的API获取 |
| 字段格式 | 日期、金额、编号等格式是否统一 |
| 排序 | 列表默认排序是否按 `task_start_time ASC` |
| 状态显示 | 状态标签的颜色和文案是否正确 |

#### 4.4 交互完整性

| 检查项 | 内容 |
|--------|------|
| 加载状态 | 数据加载时是否有 loading 提示 |
| 空状态 | 无数据时是否有空状态提示 |
| 错误处理 | API失败时是否有错误提示 |
| 操作反馈 | 增删改操作后是否有成功/失败提示 |
| 权限控制 | 非管理员/非归属机构用户是否无法访问 |

## UI开发规范（双源对照原则，本文档为唯一来源）

每个页面必须同时参考两个来源，禁止仅根据简化框架开发：

**来源1**: `开发文档/03-视觉/shejigao.pen` — 视觉设计、布局、文案
**来源2**: `开发文档/04-数据流/` — 数据模型、字段定义、业务逻辑

**开发前检查清单**:
- [ ] 读取 shejigao.pen 对应页面设计
- [ ] 读取数据逻辑文档相关定义
- [ ] 提取精确颜色代码、文案内容、布局参数、组件结构
- [ ] 确认数据来源、字段定义、展示格式

**禁止行为**:
- ❌ 仅根据简化框架文档开发
- ❌ 凭经验猜测设计细节
- ❌ 跳过设计文件对比验证

**违规后果**: 停止开发 → 重新读取设计 → 修正实现

## 注意事项

- 如果设计稿中找不到对应页面，标注为"设计稿缺失，按数据逻辑文档实现"
- 如果用户提供了截图，将截图与设计稿对比（使用 `analyze_image` 工具）
- ⚠️ 图标表示与设计有轻微偏差但可接受，❌ 表示严重不一致需要修复
