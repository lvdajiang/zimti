# 上下文保存 `/save-ctx`

> **使用场景**：会话即将结束、上下文即将溢出、或用户主动要求保存时。
> **目标**：让下次新会话能无缝接续当前工作，零信息丢失。

当用户调用 `/save-ctx` 时，执行以下步骤：

---

## 步骤1：回顾本次会话

扫描本次对话，提取以下信息：

1. **完成了什么**（已交付的工作）
2. **正在做什么**（进行中、未完成的工作）
3. **发现了什么**（新发现的问题、临时决策、未记录的约束）
4. **待解决什么**（用户提到但还没处理的需求、已知的bug）

---

## 步骤2：更新记忆文件

将提取的信息写入 `~/.claude/projects/d--project-lvyou/memory/` 目录。

### 文件结构

| 文件 | 内容 | 更新频率 |
|------|------|---------|
| `session_progress.md` | 当前开发进度、已完成功能、进行中功能 | 每次会话 |
| `decisions.md` | 重要技术决策及其原因 | 新增决策时 |
| `known_issues.md` | 已知但未修复的bug和待改进项 | 新增问题时 |
| `business_rules.md` | 业务规则确认记录（凌晨4点分界线等） | 新增规则时 |

### 写入规则

- **只追加**：新信息追加到文件末尾，不要覆盖已有内容
- **带时间戳**：每条记录标注日期（`2026-04-18`）
- **精简**：每条记录控制在3行以内，关键信息优先
- **代码引用**：涉及具体文件时，写明文件路径和行号

### 模板

#### session_progress.md 新增条目格式
```markdown
## 2026-04-18
### 已完成
- [功能描述] 涉及文件: xxx.py, yyy.vue
### 进行中
- [功能描述] 当前进度: [百分比或状态]
### 待处理
- [需求描述] 来源: 用户沟通/bug报告
```

#### decisions.md 新增条目格式
```markdown
## 2026-04-18
- **决策**: [决策内容]
  - **原因**: [为什么这么做]
  - **影响**: [影响哪些文件/功能]
  - **替代方案**: [被否决的方案及原因]
```

#### known_issues.md 新增条目格式
```markdown
## 2026-04-18
- **问题**: [问题描述]
  - **位置**: [文件:行号]
  - **严重程度**: 高/中/低
  - **临时方案**: [如有]
```

#### business_rules.md 新增条目格式
```markdown
## 2026-04-18
- **规则**: [规则名称]
  - **内容**: [规则详情]
  - **来源**: [用户确认/文档参考]
  - **关联代码**: [相关文件]
```

---

## 步骤3：更新 MEMORY.md 索引

确保 `~/.claude/projects/d--project-lvyou/MEMORY.md` 索引文件指向所有记忆文件（如果不存在则创建）。

索引格式：`- [标题](文件名.md) — 一句话描述`

---

## 步骤4：检查 CLAUDE.md 是否需要更新

事件驱动更新，检查本次会话是否触发了以下条件：

| 触发条件 | 检查方法 | 动作 |
|---------|---------|------|
| 新增/删除了技能 | 对比 CLAUDE.md 技能列表与 `.claude/skills/` 目录 | 更新技能章节 |
| 新增了技术规范 | 本次会话是否提炼了新的技术规则 | 追加到技术规范章节 |
| 完成了大模块 | 项目阶段是否推进 | 更新项目状态 |
| 发现了反复踩的坑 | 本次修了同类bug多次 | 提炼为规则写入 |
| CLAUDE.md 超过 350 行 | `wc -l CLAUDE.md` | 精简，历史内容移到记忆文件 |

**如果需要更新**，向用户报告并执行更新，然后提交 git。
**如果不需要更新**，跳过此步骤。

---

## 步骤5：向用户汇报

```
💾 上下文已保存

已更新记忆文件：
- session_progress.md: 新增 N 条进度记录
- decisions.md: 新增 N 条决策记录
- known_issues.md: 新增 N 条问题记录

下次新会话将自动读取这些信息。
```

---

## 自动触发条件

除了用户手动调用，以下情况也应主动提醒保存：
- 检测到对话即将接近上下文窗口上限
- 用户说"明天继续"、"下次再说"等结束信号
- 用户连续3次发起新话题，当前话题可能被遗忘
