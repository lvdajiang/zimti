# 设计规格 `/design-spec`

> **使用场景**：`/refine` 输出需求文档后，生成页面级设计文档。
> **前置**：`/refine` 输出的需求文档
> **后置**：`/design-page`（读设计文档生成 Pencil 稿）
> **输出**：`开发文档/02-设计/{页面名}-设计规格.md`

## 用法

```
/design-spec                    # 交互式，逐步引导
/design-spec 司机订单列表页      # 带页面名称直接生成
```

## 执行流程（4步）

### 第1步：读取需求 + 确定页面类型

读取会话中 `/refine` 的输出，确认：

1. **页面名称和定位**
2. **页面类型**（决定基础布局模板）：

| 类型 | 尺寸 | 布局模板 | 参考页面ID |
|------|------|----------|-----------|
| 桌面后台页 | 1920x1080 | Header + Content | `kEXiE` |
| 移动小程序页 | 375xauto | NavBar + Tabs + Cards + FAB | `10uo7` |
| 着陆页 | 1440xauto | Hero + Sections + Footer | `XDwNK` |
| 登录页 | 375x800 | Logo + Form + Submit | `Yg1wx` |

3. **字段清单**（从需求文档提取所有需要展示/输入的字段）

### 第2步：读取设计系统

读取 `开发文档/03-视觉/设计系统规范.md`，获取：
- 59个设计变量
- 组件规格（按钮/输入框/表格/卡片/Badge/导航）
- 状态设计（hover/disabled/loading/empty）
- 布局模板

### 第3步：生成设计文档

按以下7要素模板生成页面设计文档，每个要素都要**具体到值**：

```markdown
# 页面设计规格：{页面名称}

> 生成日期：{日期}
> 需求来源：{refine输出文件}
> 页面类型：{桌面后台页/移动小程序页/着陆页/登录页}

## 1. 布局结构

描述页面的区块划分和排列方式，具体到每个区块的高度和定位。

- 页面尺寸：{1920x1080 / 375xauto / 1440xauto}
- 整体布局：{vertical / horizontal}
- 区块列表：
  - 区块A：{名称}，{高度}px，{布局方式}，{定位}
  - 区块B：...
- 嵌套关系：用缩进表示层级

## 2. 颜色映射

为页面中每个可见元素指定颜色，全部使用 `$变量名` 引用。

| 元素 | 背景色 | 文字色 | 边框色 |
|------|--------|--------|--------|
| 页面背景 | $mobile-bg-start→$mobile-bg-end | — | — |
| 卡片 | #FFFFFF | — | — |
| 标题文字 | — | $text-primary | — |
| ... | ... | ... | ... |

## 3. 字体规范

为页面中每类文字指定字体/字号/字重。

| 文字类型 | 内容 | 字体 | 字号 | 字重 | 层级 |
|---------|------|------|------|------|------|
| 页面标题 | {示例} | Space Grotesk | 20px | 700 | H2 |
| 卡片标题 | {示例} | Inter | 14px | 600 | H4 |
| 正文 | {示例} | Inter | 14px | 400 | Body |
| ... | ... | ... | ... | ... | ... |

## 4. 间距

为页面中每个容器和元素指定间距。

| 容器/元素 | 内边距 padding | 元素间距 gap | 说明 |
|----------|---------------|-------------|------|
| 页面内容区 | 20px | 16px | $space-xl, $space-lg |
| 卡片 | 20px | 12px | $space-xl, $space-md |
| ... | ... | ... | ... |

## 5. 组件规格

列出页面中使用的每个组件及其具体参数。

| 组件 | 尺寸 | 变体 | 关键参数 |
|------|------|------|---------|
| NavBar | 52px高 | 移动导航 | fill=#FFF, radius=16px, padding=0 20px |
| Tab | 40px高, 70px宽 | active/inactive | radius=12px |
| 订单卡片 | auto高 | 列表卡片 | fill=#FFF, radius=16px, padding=20px |
| 状态Badge | 28px高 | success/warning/error | radius=8px |
| FAB | 56x56 | primary-gradient | radius=28 |
| ... | ... | ... | ... |

## 6. 状态设计

为页面中每个可交互元素和每个可能的页面状态指定设计。

### 交互状态

| 元素 | hover | disabled | 说明 |
|------|-------|----------|------|
| FAB按钮 | 颜色加深+阴影 | opacity:0.5 | — |
| Tab | active变色 | — | 点击切换筛选 |
| ... | ... | ... | ... |

### 页面状态

| 状态 | 设计 | 具体内容 |
|------|------|---------|
| loading | 骨架屏 | {N}张卡片骨架，140px高，脉冲动画 |
| empty | 图标+文字+按钮 | 图标：{emoji}，文字："{提示}"，按钮："{文案}" {变体} |
| error | 错误提示+重试 | 文字："{提示}"，按钮："重试" primary md |

## 7. 响应式规则

| 规则 | 说明 |
|------|------|
| 目标尺寸 | 375px（移动）/ 1920px（桌面） |
| 滚动策略 | {上拉加载更多 / 无限滚动 / 分页} |
| 长内容处理 | {文字截断 / 展开 / 固定高度滚动} |
```

### 第4步：保存文件

将设计文档保存到：`开发文档/02-设计/{页面名}-设计规格.md`

输出确认：
```
✅ 设计规格已生成：开发文档/02-设计/{页面名}-设计规格.md
📊 覆盖 7 要素：布局 / 颜色 / 字体 / 间距 / 组件 / 状态 / 响应式
→ 下一步：/design-page（生成 Pencil 设计稿）
```

## 内含规则（强制）

1. **全部引用变量**：颜色/间距/字号必须用 `$变量名`，禁止硬编码
2. **每个值都要具体**：不允许出现"适当间距"、"合适字号"，必须写明像素值
3. **不发明新组件**：只使用设计系统规范中已定义的组件，需要新组件时先讨论
4. **状态必须完整**：loading/empty/error 三个页面状态 + 交互元素的 hover/disabled
5. **字段全覆盖**：需求文档中列出的每个字段都必须在组件规格中找到对应位置
