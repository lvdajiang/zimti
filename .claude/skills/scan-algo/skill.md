# 算法审查 `/scan-algo`

> **使用场景**：修改了调度管道相关代码后，或定期巡检时调用。
> **目标**：结构化检查5个管道服务的逻辑完整性，发现潜在bug。

## 执行方式（Subagent 模式）

本技能通过 Subagent 执行分析，不占用主对话上下文。

收到调用后，使用 Agent tool 启动分析代理（subagent_type: "general-purpose"），传入以下 prompt：

```
你是调度管道算法审查专家。请审查以下管道服务代码，按审查清单逐项检查。

## 审查范围
{根据用户参数确定：全部5个服务 或 指定的1个服务}

## 管道服务列表
| 服务 | 文件 | 职责 | 输入→输出 |
|------|------|------|----------|
| 去重 | `backend/app/services/deduplication_service.py` | 识别重复订单 | 原始订单→去重后订单 |
| 拆分 | `backend/app/services/splitting_service.py` | 拆分多人订单 | 去重订单→拆分后任务 |
| 拼单 | `backend/app/services/combination_service.py` | 合并可拼任务 | 拆分任务→拼单组 |
| 衔接 | `backend/app/services/connection_service.py` | 构建衔接链 | 拼单组→衔接链 |
| 派单 | `backend/app/services/dispatch_service.py` | 分配司机 | 衔接链→派单结果 |

## 审查清单（每个服务必须逐项检查）

### A. 排序规则（黄金法则）
- 读取排序：数据读取后是否按 task_start_time ASC 排序？
- 输出排序：结果输出前是否再次按 task_start_time ASC 排序？
- 排序方式：是否使用 datetime.fromisoformat() 对象排序？（禁止字符串lambda排序）

### B. 日期格式
- 所有日期字段是否统一为 YYYY-MM-DD HH:MM:SS？
- 是否调用了 normalize_datetime() 或等效的格式化方法？
- 是否有未处理的日期格式边界（如空值、只传日期不传时间）？

### C. 数据完整性
- 空列表输入时是否安全返回（不报错）？
- 单条数据输入时是否正确处理？
- 极端数据（同一天100条任务、时间完全相同）是否正确处理？

### D. 业务规则
- 凌晨4点分界线：是否按 task_start_time 的4:00 AM 划分工作日？
- S类/P类客户：拼单时是否正确区分？同类才拼单？
- 人数上限：拼单后总人数是否检查 max_passengers？
- 地点约束：拼单/衔接时起终点距离是否在合理范围内？

### E. 错误处理
- 数据库查询失败是否有明确错误信息（非500）？
- UNIQUE约束冲突是否有友好的处理？
- 前端传来的脏数据（空字符串、None、错误类型）是否有防御？

### F. API端点
- 对应的 API endpoint 是否存在？
- API 的输入验证是否充分？
- API 返回的数据格式是否与前端期望一致？

## 输出格式

对每个审查的服务，输出：

🔍 算法审查报告 — [服务名]

### 排序规则
- 读取排序：[✅/❌] 说明
- 输出排序：[✅/❌] 说明
- 排序方式：[✅/❌] 说明

### 日期格式 / 数据完整性 / 业务规则 / 错误处理
- [✅/❌] 说明

### 发现的问题
| # | 严重程度 | 位置 | 问题描述 | 建议修复 |
|---|---------|------|---------|---------|
| 1 | 高/中/低 | 文件:行号 | xxx | xxx |

### 总结
[服务整体评估：健壮/基本可用/需要重构]
```

**重要**：只返回审查报告摘要给主对话，不要返回分析过程。

## 用法

```
/scan-algo                    # 审查全部5个管道服务
/scan-algo dedup              # 仅审查去重服务
/scan-algo combine            # 仅审查拼单服务
/scan-algo connect            # 仅审查衔接服务
/scan-algo dispatch           # 仅审查派单服务
/scan-algo split              # 仅审查拆分服务
```

| **触发** | 主动巡检/修改后审查 | 用户报告具体bug |

| **目标** | 预防bug | 修复bug |
