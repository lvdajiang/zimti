# 项目进度 `/status`

> **使用场景**：想快速了解项目当前状态，不需要打开代码。
> **目标**：10秒内给出全貌。

## 执行方式（Subagent 模式）

本技能通过 Subagent 执行，不占用主对话上下文。收到调用后，使用 Agent tool 启动代理（subagent_type: "general-purpose"），让代理收集 git 状态、记忆文件、文件系统信息并汇总。**只返回进度摘要给主对话。**

## 用法

```
/status                  # 查看整体进度
/status detail           # 查看详细进度
/status bugs             # 只看已知问题
/status pages            # 只看页面完成情况
```

## 执行步骤

### 第1步：收集信息（自动，不问用户）

从以下来源收集信息：

| 来源 | 获取内容 | 方式 |
|------|---------|------|
| `memory/session_progress.md` | 开发进度 | 读取文件 |
| `memory/known_issues.md` | 已知问题 | 读取文件 |
| `memory/decisions.md` | 技术决策 | 读取文件 |
| `git log --oneline -10` | 最近提交 | git 命令 |
| `git diff --stat` | 未提交改动 | git 命令 |
| `frontend/src/views/` 目录 | 已创建的页面 | 文件扫描 |
| CLAUDE.md 37个页面清单 | 页面总览 | 读取文件 |

### 第2步：输出进度报告

```

### 页面进度（X/37完成）

- [功能2] — 2026-04-17

- [需求/问题1]
```

## 与其他 skill 的关系

| Skill | 关系 |
|-------|------|
| `/status` | 只读查看，不做任何修改 |
| `/save-ctx` | 写入进度（`/status` 读取的数据来源之一） |
| `/load-ctx` | 恢复完整上下文（`/status` 是轻量版） |

## 注意事项

- `/status` 不修改任何文件，是纯只读操作
- 页面完成情况通过扫描 `frontend/src/views/` 目录下的文件判断
- 如果记忆文件不存在，仍然可以从 git 和文件系统获取基本信息
