# 页面描述审查 `/design-review`

> **使用场景**：所有 `*-页面描述.md` 写完或修改后，检查文档间一致性。
> **目标**：发现跨页面参数传递不匹配、样式变量引用错误、功能映射遗漏三类问题。
> **前置**：`/design-detail` 生成的页面描述文件

## 执行方式（Subagent 模式）

本技能通过 Subagent 执行分析，不占用主对话上下文。

**必须使用 Agent tool** 启动分析代理，传入以下参数：
- subagent_type: "general-purpose"
- prompt: 包含下方完整审查清单

代理在独立上下文中读取所有页面描述文件，交叉比对，**只返回问题清单摘要给主对话**。禁止在主窗口中逐个读取页面描述文件。

## 用法

```
/design-review                # 审查所有页面描述
/design-review 选题工作台      # 审查单个页面描述（含跨页面关联）
```

## 执行步骤

### 第1步：收集文件（10秒）

扫描 `开发文档/02-设计/*-页面描述.md`，确定审查范围。
同时读取 `开发文档/03-视觉/设计系统规范.md` 作为样式变量基准。

### 第2步：四维度审查（60秒）

#### 2.1 跨页面参数传递一致性

从每个页面的「第7章 跨页面关联」提取：
- **传出参数**：A页面传给B页面的参数名和类型
- **接收参数**：B页面从A页面接收的参数名和类型

**检查规则**：

| 检查项 | 说明 |
|--------|------|
| 参数名匹配 | A传出 `hotspot_id`，B接收必须也是 `hotspot_id` |
| 参数类型一致 | A传出数组 `video_ids[]`，B接收不能是单值 `video_id` |
| 双向覆盖 | A声称跳转B并传参，B的关联表里必须有A作为来源 |
| 缺失接收 | B页面使用某参数但关联表中未声明来源 |
| 孤立传出 | A页面声称传参给B，但B页面完全不接收 |

#### 2.2 样式变量引用合法性

读取 `开发文档/03-视觉/设计系统规范.md` 提取合法变量清单（颜色、字号、字重、间距、圆角、阴影）。

**检查每个页面描述中引用的 `$变量`**：

| 检查项 | 说明 |
|--------|------|
| 变量存在 | 引用的 `$brand-indigo` 是否在设计系统规范中有定义 |
| 值一致 | 页面描述中写的 `#6366F1` 是否与变量定义值一致 |
| 颜色格式 | 是否统一使用 hex 格式或 rgba |
| 未引用变量 | 页面描述中硬编码的色值是否有对应变量应替换 |

#### 2.3 功能映射完整性

读取每个页面对应的 `*-设计规格.md` 附录中的字段全覆盖清单，与页面描述第3章映射表交叉核对。

**检查规则**：

| 检查项 | 说明 |
|--------|------|
| 显示字段全覆盖 | 设计规格列出的每个显示字段，映射表中必须有对应行 |
| 操作全覆盖 | 设计规格列出的每个操作，映射表中必须有对应行 |
| 编号可定位 | 映射表引用的「区块#编号」在实际区块拆解中必须存在 |
| 装饰元素标注 | 没有映射到功能的元素必须标注"装饰" |
| 无多余元素 | 映射表中引用了不存在的区块编号 |

#### 2.4 跨阶段回溯对齐

从当前阶段（页面描述）向前回溯到需求文档和设计规格，验证端到端一致性。

**检查规则**：

| 检查项 | 说明 |
|--------|------|
| 字段名对齐 | 页面描述中的每个字段名是否与需求文档和设计规格中的定义一致（如同一个字段不叫 `account_name` 又叫 `accountName`） |
| 枚举值对齐 | 状态、类型、平台等枚举值在三层文档中是否一致（如需求定义"监控中/已暂停"，页面描述不能变成"运行中/已停止"） |
| 操作有需求依据 | 页面描述中的每个操作能追溯到需求文档的用户故事或操作定义，无自行增加的操作 |
| 跨页面参数双向对称 | A页面声称传参给B，B的入参定义必须匹配（名称+类型），且反向声明来源 |

**基准文件**：`开发文档/01-需求/*-需求.md` + `开发文档/02-设计/*-设计规格.md`

### 第3步：输出审查报告

```
📋 页面描述审查报告

审查范围：[N个页面描述]
基准文件：设计系统规范.md

### 汇总

| 维度 | ✅通过 | ⚠️警告 | ❌严重 |
|------|--------|--------|--------|
| 跨页面参数传递 | N | N | N |
| 样式变量引用 | N | N | N |
| 功能映射完整性 | N | N | N |
| 跨阶段回溯对齐 | N | N | N |

### 跨页面参数传递问题

| # | 严重程度 | 页面A | 页面B | 问题描述 |
|---|---------|-------|-------|---------|
| 1 | ❌ | 选题工作台 | 脚本编辑器 | A传出 content_skeleton，B接收为 content_structure，名称不匹配 |
| 2 | ⚠️ | 热点追踪 | 选题工作台 | A传出 hotspot_ids[]（数组），B描述为 hotspot_id（单值），类型不匹配 |

### 样式变量引用问题

| # | 严重程度 | 页面 | 变量/值 | 问题描述 |
|---|---------|------|--------|---------|
| 1 | ❌ | 热点追踪 | $brand-error | 变量不存在，应为 $status-error |
| 2 | ⚠️ | 人设配置 | #FF2A41 | 硬编码色值，已有变量 $xiaohongshu-red 可替换 |

### 功能映射问题

| # | 严重程度 | 页面 | 问题描述 |
|---|---------|------|---------|
| 1 | ❌ | 视频详情 | 映射表引用"区块E#8"，但区块E只到#7，编号越界 |
| 2 | ⚠️ | 爆款视频库 | 设计规格定义了"收藏"操作，映射表中无对应元素 |

### 修复建议
1. [具体修复方案]
2. [具体修复方案]
```

## 与其他 skill 的关系

| Skill | 审查对象 | 阶段 |
|-------|---------|------|
| `/spec-review` | 设计规格文档 | 设计规格写完后 |
| `/design-review` | 页面描述文档 | 页面描述写完后 |
| `/check-page` | 前端实现 vs 设计稿 | 编码完成后 |
| `/code-review` | 代码质量 | 编码完成后 |

## 严重程度定义

| 标记 | 含义 | 示例 |
|------|------|------|
| ❌ 严重 | 会导致开发对接出错 | 参数名不匹配、变量不存在、映射编号越界 |
| ⚠️ 警告 | 不影响功能但不规范 | 硬编码色值可替换为变量、类型描述模糊 |
| 💡 建议 | 可优化项 | 冗余的装饰元素、文案可以更统一 |

## 注意事项

- 审查基于文档间交叉比对，不涉及代码文件
- 如果某个页面描述文件不存在，跳过并标注
- 设计系统规范是样式变量的唯一基准，不以其他页面描述为准
