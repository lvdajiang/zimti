# 页面详细描述 `/design-detail`

> **使用场景**：`/design-spec` 输出页面清单后，为每个页面生成详细描述。
> **前置**：`/design-spec` 输出的设计规格文件
> **后置**：`/design-page`（读页面描述生成 Pencil 稿）
> **输出**：`开发文档/02-设计/{页面名}-页面描述.md`

## 用法

```
/design-detail                    # 交互式，选择要描述的页面
/design-detail 司机订单列表页      # 带页面名称直接生成
```

## 执行流程（4步）

### 第1步：读取设计规格

读取 `开发文档/02-设计/{页面名}-设计规格.md`，提取：
- 页面类型和尺寸（决定布局模板）
- 7要素参数（颜色/字体/间距/组件/状态/响应式）
- 字段清单（从需求文档继承）

### 第2步：读取需求文档 + 数据流参考

| 输入 | 路径 | 提取什么 |
|------|------|---------|
| 需求文档 | `开发文档/01-需求/{功能名}-需求.md` | 用户故事、字段、操作、边缘场景 |
| 设计系统 | `开发文档/03-视觉/设计系统规范.md` | 组件规格、状态设计规范 |

如果该页面已有数据流文档（`开发文档/04-数据流/{页面名}-数据流.md`），也一并读取，用于字段溯源。

### 第3步：生成页面描述

按以下模板生成页面描述文档，**具体到每个元素的文案、位置、尺寸**：

```markdown
# 页面描述：{页面名称}

> 生成日期：{日期}
> 前置文档：设计规格 / 需求文档
> 页面类型：{桌面后台页/移动小程序页/着陆页/登录页}
> 尺寸：{1920x1080 / 375xauto}

## 1. 页面总览

- **页面目标**：一句话描述用户在这个页面要完成什么
- **角色权限**：哪些角色能看到这个页面
- **进入方式**：从哪些页面/操作可以进入

## 2. 区块拆解

## 3. 功能→元素映射表（完整性校验）

将 `/refine` 第3步定义的**每个显示字段和每个操作**，逐一映射到本页面的具体UI元素。确保功能不遗漏、元素不冗余。

### 3.1 显示字段映射

| refine定义的显示字段 | 实现元素（区块#编号） | 备注 |
|--------------------|---------------------|------|
| 状态 | 卡片列表/状态标签(#1) | 颜色标签 |
| 订单编号 | 卡片列表/订单编号(#2) | — |
| 路线 | 卡片列表/路线(#3) | 起止城市 |
| 时间 | 卡片列表/时间(#4) | — |
| 金额 | 卡片列表/金额(#5) | 加粗 |

### 3.2 操作映射

| refine定义的操作 | 实现元素（区块#编号） | 触发方式 | 备注 |
|-----------------|---------------------|---------|------|
| 查看详情 | 卡片列表/整个卡片 | 点击 | 跳转详情页 |
| 按状态筛选 | 筛选栏/标签组 | 点击切换 | 重置分页 |
| 下拉刷新 | 全页 | 手势下拉 | 重新加载 |
| 上拉加载 | 列表底部 | 滚动到底 | 追加数据 |

### 3.3 完整性检查

| 检查项 | 结果 |
|--------|------|
| refine的每个显示字段 → 都有对应元素？ | ✅/❌ 缺少：___ |
| refine的每个操作 → 都有对应元素？ | ✅/❌ 缺少：___ |
| 每个元素 → 都能追溯到refine功能？ | ✅/❌ 多余：___ |

**规则**：
- 有功能没元素 = 遗漏，必须补充元素
- 有元素没功能 = 多余，必须标注用途或删除
- 纯装饰元素（分割线、背景图）标注"装饰"，不需要映射功能
```

## 内含规则（强制）

1. **具体到每个元素**：每个可见元素都要有文案、尺寸、样式变量、数据绑定，不允许出现"类似XX"的模糊描述
2. **功能映射必须完整**：第3章的功能→元素映射表必须100%覆盖refine定义的所有显示字段和操作。有功能没元素=遗漏，有元素没功能=多余
3. **溯源标记保留**：数据绑定的字段标注 📥存储 / 🔗调用 / 📊计算 / ⛔禁止
4. **文案必须写定**：按钮文字、提示文字、空状态文案必须写明具体内容，不允许"适当文案"
5. **操作定义不重复业务逻辑**：第4章操作定义只描述视觉反馈和交互细节，业务行为在refine已定义，通过映射表引用
6. **不写实现代码**：只描述"是什么"，不描述"怎么做"（那是编码阶段的事）
7. **严格继承设计规格**：7要素参数以设计规格为准，不自行修改
8. **状态必须完整**：loading / empty / error 三个页面状态 + 各筛选条件的差异化空状态
