# 恢复上下文 `/load-ctx`

> **使用场景**：新会话开始时，需要接续上次的工作。
> **目标**：零信息丢失，无缝接续。

## 用法

```
/load-ctx                # 读取所有记忆文件，汇报当前状态
/load-ctx progress       # 只看开发进度
/load-ctx issues         # 只看已知问题
/load-ctx decisions      # 只看技术决策
```

## 执行步骤

### 第1步：读取记忆文件

从 `~/.claude/projects/d--project-lvyou/memory/` 目录读取以下文件：

| 文件 | 内容 |
|------|------|
| `session_progress.md` | 开发进度（已完成、进行中、待处理） |
| `decisions.md` | 技术决策及其原因 |
| `known_issues.md` | 已知但未修复的问题（如存在） |
| `business_rules.md` | 业务规则确认记录（如存在） |

同时读取 `MEMORY.md` 索引，检查是否有其他相关记忆文件。

### 第2步：汇报当前状态

```
📋 上下文恢复

### 上次进度（2026-04-18）
已完成：
- [功能1] 涉及文件: xxx
- [功能2] 涉及文件: yyy

进行中：
- [功能3] 当前进度: 50%

待处理：
- [需求1] 来源: 用户沟通
- [bug1] 位置: xxx:L42

### 重要决策
- [决策1]: [原因]

### 已知问题
- [问题1]: [严重程度]

### 建议下一步
1. [建议1]
2. [建议2]
```

### 第3步：确认接续方向

根据上次会话的"进行中"和"待处理"内容，向用户确认接下来要做什么：

```
上次有待处理的工作，你想：
1. 继续上次的 [功能X]
2. 处理已知问题 [问题Y]
3. 做新的任务
```

## 自动触发条件

以下情况应主动执行 `/load-ctx`（不需要用户手动调用）：
- 用户说"继续"、"接着做"、"上次"
- 用户提到之前会话中的工作内容
- 新会话开始且用户没有给出明确的新任务

## 与 `/save-ctx` 的关系

```
/save-ctx  →  写入记忆文件
/load-ctx  →  读取记忆文件
```

每次会话结束时用 `/save-ctx` 保存，新会话开始时用 `/load-ctx` 恢复，形成闭环。
