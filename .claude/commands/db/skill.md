# 数据库快捷查询 `/db`

> **使用场景**：需要查看数据库中的数据时。
> **目标**：通过 SQLite MCP 工具直接查库，不绕 API。

当用户调用 `/db` 时，执行以下步骤。

---

## 步骤1：理解查询意图

从用户的自然语言中提取查询要素：
- **查什么**：表名、字段、条件
- **要什么**：计数、明细、统计、对比

**常见意图映射**：

| 用户说 | 实际操作 |
|:-------|:---------|
| "有多少条数据" | `SELECT COUNT(*) FROM 表名` |
| "看看今天的数据" | `SELECT * FROM 表名 WHERE date(created_at) = date('now')` |
| "有哪些景区" | `SELECT * FROM entities WHERE entity_type = '景区供应'` |
| "表结构是什么" | `describe_table(表名)` |
| "数据库里有哪些表" | `list_tables()` |
| "查一下最近的订单" | `SELECT * FROM orders ORDER BY created_at DESC LIMIT 20` |

## 步骤2：执行查询

**优先使用 MCP 工具**：
1. `list_tables()` — 列出所有表和行数
2. `describe_table(表名)` — 查看表结构
3. `query_sql(SQL)` — 执行只读查询

**查询规范**：
- 必须使用只读查询（SELECT）
- 结果超过 50 行时自动截断并提示
- 时间字段统一用 `YYYY-MM-DD` 格式
- 涉及日期筛选时，确认 SQLite 日期函数格式

## 步骤3：格式化输出

查询结果以表格形式呈现：
```
表名: resources（63180 行）
┌──────────┬────────────┬──────────┐
│ id       │ name       │ type     │
├──────────┼────────────┼──────────┤
│ RES0001  │ 乌鲁木齐XX │ 房       │
│ RES0002  │ 喀纳斯XX   │ 房       │
└──────────┴────────────┴──────────┘
共 63180 行
```

同时提供分析摘要：
- 数据分布情况
- 异常值提示
- 与业务相关的观察

---

## 注意事项

- 只查 `zhipai.db`，不查备份文件
- 不执行任何写操作（INSERT/UPDATE/DELETE）
- 如果用户要求修改数据，提示"请通过 API 或管理页面操作"
- 如果表名不确定，先用 `list_tables()` 确认
